## 導入

Tutorial 1では、プログラミングの基本的な考え方や、学習に臨む心構えについて学びました。いよいよ、実際にコードを書いていく準備を始めましょう！

その最初のステップが「**開発環境の構築**」です。

「環境構築」と聞くと、何やら難しそうな専門用語に聞こえるかもしれません。「黒い画面に謎の呪文を打ち込むのでは…」と不安に思う方もいるでしょう。しかし、心配は無用です。一つ一つのツールの役割を理解すれば、決して難しいことではありません。

### 比喩で理解する: 開発環境は「料理人のためのキッチン」

あなたは、プロの料理人（プログラマー）を目指しています。最高の料理（プログラム）を作るためには、まず何が必要でしょうか？

そうです、「**キッチン（開発環境）**」です。

*   食材を切るための「**包丁とまな板**」（テキストエディタ）
*   調理するための「**コンロやオーブン**」（サーバ環境）
*   出来上がった料理を味見するための「**お皿とフォーク**」（Webブラウザ）

これらの道具が揃っていなければ、どんなに素晴らしいレシピ（アイデア）を持っていても、料理を始めることすらできませんよね。

開発環境の構築とは、まさにこの「**自分専用のキッチンを準備する作業**」なのです。プログラミングという料理を快適に、そして効率的に行うための、様々な道具を自分のコンピュータにインストールし、設定していく。それが環境構築です。

このセクションでは、これから私たちが一緒に準備していく「キッチン」に、どんな道具があって、それぞれがどんな役割を持っているのかを見ていきましょう。

## 概念の説明

### 🔑 なぜ「自分のPC」に環境構築が必要なのか？

「プログラミングって、Webサイト上でできるサービスもあるんじゃないの？」と思う方もいるかもしれません。確かに、簡単なコードを試すだけなら、そうしたサービスは非常に便利です。

しかし、本格的なWebアプリケーションを開発するとなると、話は別です。なぜなら、Webアプリケーションは、「**サーバ**」と「**クライアント**」という、2つの異なるコンピュータが連携して動作する仕組みだからです。

| 登場人物 | 役割 | 具体例 |
| :--- | :--- | :--- |
| **クライアント (Client)** | サービスを**利用する側**のコンピュータ。 | あなたのPCやスマートフォン |
| **サーバ (Server)** | サービスを**提供する側**のコンピュータ。 | Webサイトのデータを保管している、24時間稼働の高性能コンピュータ |

あなたが普段、Webサイトを見ている時、あなたのPC（クライアント）は、インターネットを通じてWebサイトのサーバに「このページを見せてください」と**リクエスト（要求）**を送り、サーバは「はい、どうぞ」とページのデータをあなたのPCに**レスポンス（応答）**として返しています。この一連の流れを**クライアントサーバモデル**と呼びます。

私たちがこれから作ろうとしているのは、この「サーバ側で動くプログラム」です。そのため、自分のPCの中に、サーバの動きをシミュレーションするための「**疑似的なサーバ環境**」を作る必要があるのです。これが、開発環境を構築する大きな理由の一つです。

### 🔑 このチャプターで揃える開発ツール

このチャプターでは、数ある開発ツールの中から、現代のWeb開発で最も標準的とされる、以下のツールをインストールしていきます。

1.  **Webブラウザ (Google Chrome)**
    *   **役割**: ユーザーが実際に目にする画面を表示し、動作を確認する。
    *   **キッチンの例え**: 出来上がった料理を味見するための「お皿とフォーク」。

2.  **テキストエディタ (Visual Studio Code)**
    *   **役割**: プログラムのコード（レシピ）を書くための、高機能なメモ帳。
    *   **キッチンの例え**: 食材を切ったり下ごしらえをしたりする「包丁とまな板」。

## 詳細解説

### ⚙️ 1. Webブラウザ: Google Chrome

Webブラウザは、皆さんが普段インターネットを見るときに使っているお馴染みのソフトです。`Google Chrome`、`Safari`、`Firefox`、`Microsoft Edge`など、様々な種類があります。

**なぜ開発にChromeが良いのか？**

どのブラウザでもWebサイトを見ることはできますが、Web開発者は`Google Chrome`を選ぶことが多いです。その最大の理由は、「**デベロッパーツール（開発者ツール）**」が非常に高機能だからです。

デベロッパーツールは、Webページの裏側を覗き見ることができる、魔法のような機能です。

*   ページのHTMLやCSSがどのように書かれているかを確認できる。
*   サーバとの通信内容（リクエストとレスポンス）を監視できる。
*   プログラムのエラー（バグ）の原因を特定するためのヒントを得られる。

このデベロッパーツールを使いこなせるかどうかで、開発の効率は天と地ほどの差が出ます。このコースでも、デバッグ（エラー修正）の際には頻繁に利用することになります。

### ⚙️ 2. テキストエディタ: Visual Studio Code (VSCode)

プログラムのコードは、極端な話、Windowsの「メモ帳」やMacの「テキストエディット」のような、標準のテキストファイル編集ソフトでも書くことができます。

しかし、プロの料理人が家庭用の包丁ではなく、切れ味の良い専門の包丁を使うように、プロのエンジニアは「**コーディングに特化した高機能なテキストエディタ**」を使います。その中でも、現在、世界中の開発者から絶大な支持を得ているのが、Microsoftが開発した**`Visual Studio Code`（通称: VSCode）**です。

**なぜVSCodeが良いのか？**

*   **無料**: 高機能でありながら、完全に無料で利用できます。
*   **軽い**: 動作が非常に軽快で、ストレスなくコーディングに集中できます。
*   **賢い**: コードの入力補完（シンタックスハイライト、サジェスト機能）が強力で、タイプミスを防ぎ、開発スピードを向上させてくれます。
*   **拡張性が高い**: 「拡張機能（エクステンション）」を追加することで、自分好みに機能をどんどんカスタマイズできます。例えば、PHPを書くための便利な機能を追加したり、見た目を好きなデザインに変えたりすることが可能です。

このコースでは、VSCodeを単なるメモ帳としてではなく、開発の相棒として使いこなしていくための、便利な設定や拡張機能も紹介していきます。
