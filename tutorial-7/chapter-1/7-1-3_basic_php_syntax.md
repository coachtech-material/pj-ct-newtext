## 🎯 このセクションで学ぶこと

*   PHPコードを記述するための、基本的なルール（構文）を理解する。
*   コメントの書き方と、その重要性を学ぶ。
*   画面に文字を出力するための、`echo`の使い方をマスターする。
*   文の終わりを示す、セミコロン（`;`）の役割を理解する。

## 導入

プログラミング言語を学ぶことは、外国語を学ぶことと、よく似ています。単語（関数や変数）を覚えるだけでなく、それらを、どのように並べれば、正しい文章（プログラム）になるのか、という、**文法（Syntax）**を、理解する必要があります。

文法を間違えると、コンピュータは、私たちの意図を、全く理解してくれません。幸いなことに、PHPの基本的な文法は、非常にシンプルで、直感的です。

このセクションでは、PHPのプログラムを構成する、最も基本的な、3つの要素、「コメント」「`echo`による出力」「セミコロン」について、学んでいきましょう。これらは、これから、あなたが書く、全てのPHPコードの、基礎となる、重要なルールです。

---

## 詳細解説

### 🧱 PHPコードの基本構造

PHPのコードは、`<?php` という開始タグで始まり、`?>` という終了タグで終わるブロックの中に、記述します。ただし、ファイルの末尾が、PHPコードで終わる場合は、終了タグ `?>` は、**省略することが推奨**されています。これは、ファイルの末尾に、意図しない空白などが、含まれることを防ぐためです。

```php
<?php

// ここにPHPのコードを記述します

// ファイルの最後なので、終了タグは省略します
```

### 🗣️ コメント (Comments)

コメントは、プログラムの動作には、一切影響を与えない、**人間向けのメモ書き**です。コードが、何をしているのかを説明したり、後で修正するべき箇所を、メモしたりするために使用します。

複雑なコードや、他人が書いたコードを、理解するためには、コメントが、非常に重要な手がかりとなります。良いコメントを書くことは、良いコードを書くことと、同じくらい、大切なスキルです。

PHPでは、2種類のコメントの書き方があります。

1.  **一行コメント**: `//` から行末までが、コメントになります。短い説明や、一行単位のメモに適しています。

    ```php
    // これは一行コメントです。
    echo "Hello, World!"; // この行は、画面に文字を出力します。
    ```

2.  **複数行コメント**: `/*` で始まり、`*/` で終わる範囲が、全てコメントになります。複数行にわたる、長い説明や、一時的に、コードのブロックを、丸ごと無効化したい場合（コメントアウト）に便利です。

    ```php
    /*
     * これは複数行コメントです。
     * 関数の説明などを、詳しく書くときに使います。
     * @param string $name 名前
     * @return string 挨拶文
     */
    function greet($name)
    {
        return "こんにちは、" . $name . "さん！";
    }

    /*
    // 一時的に、このコードを無効にしたい
echo "この部分は実行されません";
    */
    ```

> **💡 なぜコメントは重要か？**
> 未来の自分は、他人です。数週間後に、自分の書いたコードを見返したとき、「なぜ、こんな書き方をしたんだっけ？」と、頭を抱えることは、日常茶飯事です。コメントは、未来の自分や、チームの他のメンバーへの、親切な「道しるべ」なのです。

### 📺 画面への出力 (echo)

`echo`は、指定した文字列や、変数の内容を、画面（HTML）に出力するための、最も基本的な命令です。PHPでは、`echo`は、言語構造（language construct）と呼ばれる、特別な命令であり、関数ではありませんが、最初は、「画面に文字を出すための命令」と覚えておけば、問題ありません。

文字列は、シングルクォーテーション（`'`）または、ダブルクォーテーション（`"`）で囲みます。

```php
<?php

// 文字列を直接出力
echo "Hello, PHP!";

echo '<p>HTMLタグも、そのまま出力できます。</p>';

// 数値や計算結果も出力可能
echo 123;
echo 10 + 5; // 結果の「15」が出力される
```

> **シングルクォーテーション vs ダブルクォーテーション**
> 2つの違いは、**変数展開**と**エスケープシーケンス**の扱いです。今の時点では、難しく考えず、「**基本はシングルクォーテーションを使い、変数展開が必要なときだけ、ダブルクォーテーションを使う**」と覚えておくと良いでしょう。詳細は、変数のセクションで、詳しく学びます。

### 🔚 文の終わり (Semicolon)

PHPでは、原則として、**各命令（文）の終わり**に、**セミコロン（`;`）**を付ける必要があります。これは、コンピュータに対して、「ここで、一つの命令が終わりですよ」と、明確に、伝えるための、区切り記号です。

```php
<?php

echo "最初の文";
echo "次の文"; // セミコロンを忘れると、エラーになる！

$today = "Monday";
echo $today;
```

プログラミング初学者が、最も、遭遇しやすいエラーの一つが、この、セミコロンの付け忘れです。エラーメッセージで、`syntax error, unexpected ...` のような、文法エラーを指摘された場合は、まず、直前の行の、セミコロンを、確認する癖をつけましょう。

## ✨ まとめ

このセクションでは、PHPのプログラムを記述するための、基本的な文法ルールを学びました。

*   **PHPブロック**: コードは、`<?php ... ?>` の中に記述する。ファイルの最後では、`?>` は省略する。
*   **コメント**: 人間向けのメモ。`//` で一行コメント、`/* ... */` で複数行コメント。
*   **`echo`**: 画面に、文字列や、変数の内容を、出力する命令。
*   **セミコロン (`;`)**: 各命令の終わりに、必ず付ける、区切り記号。

これらのルールは、非常にシンプルですが、PHPプログラミングの、根幹をなすものです。特に、コメントを書く習慣と、セミコロンを忘れない意識は、今のうちから、しっかりと、身につけておきましょう。
