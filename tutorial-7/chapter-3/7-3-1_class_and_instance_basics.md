# Tutorial 7-3-1: クラスとインスタンスの基礎

## 🎯 このセクションで学ぶこと

*   オブジェクト指向プログラミング（OOP）の、基本的な考え方を理解する。
*   クラスとは何か（設計図）、インスタンスとは何か（実体）を、比喩を使って理解する。
*   PHPで、`class`キーワードを使って、クラスを定義する方法を学ぶ。
*   `new`キーワードを使って、クラスから、インスタンスを生成する方法を学ぶ。

---

## 導入

これまでは、データ（変数）と、処理（関数）を、別々に扱ってきました。しかし、プログラムが、複雑になってくると、「このデータと、この処理は、密接に関連しているのに、なぜ、バラバラに、管理しなければならないのだろう？」という、疑問が湧いてきます。

例えば、「ユーザー」という概念を考えてみましょう。「ユーザー」には、名前や、年齢といった、**データ**があります。そして、「自己紹介する」という、**処理（振る舞い）**があります。これらは、明らかに、一つのまとまりとして、扱えると、便利そうです。

この、「**データと、それを操作する処理を、一つのまとまり（オブジェクト）として、カプセル化する**」という考え方が、**オブジェクト指向プログラミング (Object-Oriented Programming, OOP)** の、基本的なアイデアです。OOPは、大規模で、複雑なソフトウェアを、整理整頓し、見通し良く、開発するための、強力な、設計思想です。

このチャプターでは、Laravelを、使いこなす上で、最低限、必要な、OOPの、エッセンスである、「クラス」と「インスタンス」について、学んでいきます。

---

## 詳細解説

### 🏛️ クラスとインスタンス: 設計図と実体

オブジェクト指向の、最も、中核となる概念が、**クラス (Class)** と、**インスタンス (Instance)** です。この2つの関係は、よく、**「たい焼きの型」と「たい焼き」**に例えられます。

*   **クラス (Class)**: **設計図**や、**型**に相当します。たい焼きの例で言えば、「たい焼きの型」そのものです。クラスは、「たい焼きは、あんこを持っていて、頭から食べられる」といった、**共通の、性質（データ）と、振る舞い（処理）**を定義します。クラス自体は、まだ、具体的な、たい焼きではなく、あくまで、設計情報です。

*   **インスタンス (Instance)**: **設計図から作られた、実体**です。たい焼きの例で言えば、「たい焼きの型から作られた、個々の、たい焼き」です。インスタンスは、クラスという、設計図に基づいて、作られますが、それぞれが、独立した、存在です。例えば、あるたい焼きは、つぶあん、別のたい焼きは、こしあん、といったように、個別の、状態を持つことができます。

オブジェクト指向プログラミングでは、まず、**クラスという、設計図を定義**し、次に、その**クラスから、インスタンス（オブジェクトとも呼ばれる）を、生成して、利用する**、という手順で、プログラムを構築していきます。

### ✍️ クラスの定義

PHPで、クラスを定義するには、`class`キーワードを使います。クラス名の、最初の文字は、大文字にするのが、慣習です（アッパーキャメルケース）。

```php
// Userという名前のクラスを定義する
class User
{
    // この中に、プロパティ（データ）と、メソッド（処理）を定義していく
}
```

この時点では、クラスは、まだ、空っぽの設計図です。中身は、次のセクションで、定義していきます。

### 🆕 インスタンスの生成

クラスから、インスタンスを生成するには、`new`キーワードを使います。

```php
// Userクラスの設計図から、2つのインスタンス（実体）を生成する
$user1 = new User();
$user2 = new User();

// var_dump()で、インスタンスの中身を見てみる
var_dump($user1);
// object(User)#1 (0) {}

var_dump($user2);
// object(User)#2 (0) {}
```

*   `$user1 = new User();` というコードは、「`User`クラスの、新しいインスタンスを生成し、それを、変数`$user1`に代入せよ」という意味です。
*   `var_dump()`の結果を見ると、`$user1`と`$user2`は、どちらも`User`クラスの、オブジェクト（インスタンス）ですが、`#1`と`#2`のように、別々の、実体として、認識されていることがわかります。

これで、`User`という、概念を、プログラムの世界に、持ち込むことができました。しかし、まだ、このユーザーは、名前もなければ、何もできません。次のセクションでは、この`User`クラスに、具体的な、データ（プロパティ）と、処理（メソッド）を、追加していきます。

## ✨ まとめ

このセクションでは、オブジェクト指向プログラミングの、入り口である、クラスと、インスタンスについて学びました。

*   **オブジェクト指向 (OOP)**: データと、それを操作する処理を、一つのまとまり（オブジェクト）として、扱う、プログラミングの考え方。
*   **クラス (Class)**: オブジェクトの、**設計図**。共通の、性質と、振る舞いを定義する。
*   **インスタンス (Instance)**: クラスから作られた、**実体**。オブジェクトとも呼ばれる。
*   **クラスの定義**: `class クラス名 { ... }` の形式で、定義する。
*   **インスタンスの生成**: `new クラス名()` の形式で、生成する。

「クラスは設計図、インスタンスは実体」という、この関係性を、しっかりと、イメージできるようになることが、オブジェクト指向を、理解するための、第一歩です。Laravelの、内部では、数多くの、クラスが定義されており、私たちは、それらのクラスから、インスタンスを生成し、利用することで、アプリケーションを、構築していくことになります。
