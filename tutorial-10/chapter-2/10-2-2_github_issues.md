## 導入：GitHubイシューとは何か？

**GitHubイシュー**とは、**GitHubのリポジトリで課題やタスクを管理するための機能**です。

イシューを使うと、以下のことができます。

*   新機能の提案
*   バグ報告
*   質問
*   タスク管理

---

## 詳細解説

### 🔧 イシューの作成

#### ステップ1: GitHubリポジトリにアクセス

GitHubリポジトリのページを開きます。

---

#### ステップ2: 「Issues」タブをクリック

---

#### ステップ3: 「New issue」ボタンをクリック

---

#### ステップ4: イシューの内容を入力

**タイトル**: ログイン機能の実装

**説明**:

```
## 概要
ユーザーがログインできる機能を実装する

## 詳細
- メールアドレスとパスワードでログインできる
- ログインに成功したら、ダッシュボードにリダイレクトする
- ログインに失敗したら、エラーメッセージを表示する

## タスク
- [ ] ログインフォームの作成
- [ ] ログイン処理の実装
- [ ] バリデーションの実装
- [ ] テストの作成
```

---

#### ステップ5: 「Submit new issue」ボタンをクリック

### 💡 TIP: Markdownを使った記述

GitHubイシューでは、**Markdown記法**を使って、見やすく記述できます。

*   `##`: 見出し
*   `-`: リスト
*   `- [ ]`: チェックボックス
*   `` ` ``: コード
*   ` ``` `: コードブロック

### 🔧 イシューテンプレートの作成

イシューテンプレートを作成すると、**イシューの作成時に自動的にテンプレートが表示**されます。

#### ステップ1: `.github/ISSUE_TEMPLATE`ディレクトリを作成

```bash
mkdir -p .github/ISSUE_TEMPLATE
```

---

#### ステップ2: `bug_report.md`ファイルを作成

**`.github/ISSUE_TEMPLATE/bug_report.md`**

```markdown
---
name: Bug report
about: Create a report to help us improve
title: '[BUG] '
labels: bug
assignees: ''
---

## 概要
<!-- バグの概要を記述してください -->

## 再現手順
1. 
2. 
3. 

## 期待する動作
<!-- 期待する動作を記述してください -->

## 実際の動作
<!-- 実際の動作を記述してください -->

## 環境
- OS: 
- ブラウザ: 
- バージョン: 
```

---

#### ステップ3: `feature_request.md`ファイルを作成

**`.github/ISSUE_TEMPLATE/feature_request.md`**

```markdown
---
name: Feature request
about: Suggest an idea for this project
title: '[FEATURE] '
labels: enhancement
assignees: ''
---

## 概要
<!-- 新機能の概要を記述してください -->

## 詳細
<!-- 新機能の詳細を記述してください -->

## タスク
- [ ] 
- [ ] 
- [ ] 
```

### 💡 TIP: イシューのクローズ

イシューをクローズするには、以下の方法があります。

#### 方法1: イシューページで「Close issue」ボタンをクリック

---

プルリクエストの説明に`Closes #1`と書くと、マージ時に自動的にイシュー#1がクローズされます。

### 🚨 よくある間違い

#### 間違い1: イシューの説明が不十分

```
# NG
タイトル: バグ修正
説明: （なし）
```

**対処法**: イシューの説明を詳しく書きます。

```
# OK
タイトル: ログインエラーが表示されない
説明:
## 概要
ログインに失敗したときに、エラーメッセージが表示されない

## 再現手順
1. ログインフォームにアクセス
2. 間違ったメールアドレスとパスワードを入力
3. ログインボタンをクリック
```

---

#### 間違い2: ラベルを付けない

**対処法**: イシューにラベルを付けて、種類を明確にします。
