## 🎯 演習課題：自己紹介ページをレスポンシブ対応にしよう

### 課題の概要

これまで作成してきた自己紹介ページを、スマートフォンやタブレットでも見やすいように、レスポンシブ対応にしてください。

### 📋 要件

`style.css`にメディアクエリを追加して、以下の要件を満たしてください。

#### 1. ブレークポイントの設定

以下の3つのブレークポイントを設定します：

- **デスクトップ**：801px以上（デフォルト）
- **タブレット**：481px〜800px
- **スマートフォン**：480px以下

#### 2. タブレット（481px〜800px）のスタイル

- `.container`の最大幅を`90%`に変更
- `.container`のパディングを`30px`に変更
- `h1`のフォントサイズを`28px`に変更
- `h2`のフォントサイズを`20px`に変更

#### 3. スマートフォン（480px以下）のスタイル

- `body`のマージンを`10px`に変更
- `.container`のパディングを`20px`に変更
- `h1`のフォントサイズを`24px`に変更
- `h2`のフォントサイズを`18px`に変更
- `ul, ol`の左マージンを`10px`に変更

#### 4. 画像のレスポンシブ対応

すべてのデバイスで画像が親要素の幅に合わせて表示されるように、以下のスタイルを追加してください：

```css
img {
    max-width: 100%;
    height: auto;
}
```

## 💡 ヒント

詰まったときは、以下のヒントを参考にしてください。

### ヒント1: メディアクエリの基本構文

```css
@media (max-width: 800px) {
    /* タブレット以下のスタイル */
    .container {
        max-width: 90%;
    }
}
```

### ヒント2: ブレークポイントの順序

```css
/* デスクトップ（デフォルト） */
.container {
    max-width: 800px;
}

/* タブレット */
@media (max-width: 800px) {
    .container {
        max-width: 90%;
    }
}

/* スマートフォン */
@media (max-width: 480px) {
    .container {
        padding: 20px;
    }
}
```

### ヒント3: 複数のプロパティを変更

```css
@media (max-width: 480px) {
    h1 {
        font-size: 24px;
    }
    
    h2 {
        font-size: 18px;
    }
}
```

- 幅800pxの白いカード

## 🏃 実践: 一緒に作ってみましょう！

ちゃんとできましたか？レスポンシブデザインはメディアクエリの書き方が重要です。一緒に手を動かしながら、メディアクエリを追加していきましょう。

### 💭 実装の思考プロセス

レスポンシブデザインを実装する際、以下の順番で考えると効率的です：

1. **デスクトップファーストで設計**：まずデスクトップ用のスタイルを完成させる
2. **ブレークポイントを決める**：タブレットとスマホの境界を設定
3. **大きい画面から調整**：タブレット→スマホの順でメディアクエリを追加
4. **必要な要素だけ調整**：全てを変える必要はない、問題がある部分だけ修正
5. **実機でテスト**：ブラウザの開発者ツールで確認

「モバイルファースト」というアプローチもありますが、今回は既存のデスクトップデザインにメディアクエリを追加する形で進めます。

---

### 📝 ステップバイステップで実装

#### ステップ1: 画像のレスポンシブ対応（全デバイス共通）

**何を考えているか**：
- 「画像がコンテナからはみ出さないようにしたい」
- 「どのデバイスでも画像が適切なサイズで表示されるようにしよう」

まず、既存の`img`スタイルに以下が含まれているか確認します：

```css
img {
    max-width: 100%;
    height: auto;
    border-radius: 8px;
    box-shadow: 0 2px 8px rgba(0,0,0,0.1);
}
```

**コードリーディング**：

```css
    max-width: 100%;
```
→ 画像の最大幅を親要素の100%に制限します。これにより、小さい画面でも画像がはみ出しません。

```css
    height: auto;
```
→ 高さを自動調整して、画像の縦横比を維持します。画像が伸びたり潰れたりしません。

---

#### ステップ2: タブレット用のメディアクエリを追加する

**何を考えているか**：
- 「画面幅が800px以下のときに適用されるスタイルを書こう」
- 「コンテナの幅を少し狭くして、見出しも小さくしよう」

`style.css`の最後に、以下のメディアクエリを追加します：

```css
/* タブレット（481px〜800px） */
@media (max-width: 800px) {
    .container {
        max-width: 90%;
        padding: 30px;
    }
    
    h1 {
        font-size: 28px;
    }
    
    h2 {
        font-size: 20px;
    }
}
```

**コードリーディング（1行ずつ）**：

```css
@media (max-width: 800px) {
```
→ `@media`ルールでメディアクエリを開始します。`(max-width: 800px)`は「画面幅が800px以下のとき」という条件です。

```css
    .container {
        max-width: 90%;
```
→ コンテナの最大幅を800pxから90%に変更します。タブレットでは画面幅に対して相対的な幅にします。

```css
        padding: 30px;
```
→ パディングを40pxから30pxに縮小します。画面が小さいので、余白を節約します。

```css
    h1 {
        font-size: 28px;
    }
```
→ `h1`のフォントサイズを32pxから28pxに縮小します。タブレットでは少し小さい方がバランスが良いです。

```css
    h2 {
        font-size: 20px;
    }
}
```
→ `h2`のフォントサイズを24pxから20pxに縮小します。最後の`}`でメディアクエリを閉じます。

---

#### ステップ3: スマートフォン用のメディアクエリを追加する

**何を考えているか**：
- 「さらに小さい画面（480px以下）用のスタイルを追加しよう」
- 「スマホではさらにコンパクトにしたい」

タブレット用のメディアクエリの後に、以下を追加します：

```css
/* スマートフォン（480px以下） */
@media (max-width: 480px) {
    body {
        margin: 10px;
    }
    
    .container {
        padding: 20px;
    }
    
    h1 {
        font-size: 24px;
    }
    
    h2 {
        font-size: 18px;
    }
    
    ul, ol {
        margin-left: 10px;
    }
}
```

**コードリーディング**：

```css
@media (max-width: 480px) {
```
→ 画面幅が480px以下のときに適用されるスタイルです。スマートフォン専用の調整を行います。

```css
    body {
        margin: 10px;
    }
```
→ bodyのマージンを20pxから10pxに縮小します。スマホの小さい画面では、少しでもコンテンツ表示領域を広げたいからです。

```css
    .container {
        padding: 20px;
    }
```
→ コンテナのパディングをさらに20pxに縮小します。

```css
    h1 {
        font-size: 24px;
    }
    
    h2 {
        font-size: 18px;
    }
```
→ 見出しのフォントサイズをさらに縮小します。h1は24px、h2は18pxにします。

```css
    ul, ol {
        margin-left: 10px;
    }
}
```
→ リストの左マージンを20pxから10pxに縮小します。スマホでは横幅が限られているため、インデントを小さくします。

#### ステップ4: ブラウザの開発者ツールで確認する

**何を考えているか**：
- 「実際に異なる画面サイズで表示して確認しよう」
- 「ブレークポイントで適切にスタイルが切り替わるか確認しよう」

Chromeの開発者ツール（F12）を開いて、以下を確認します：

1. **デバイスツールバーをクリック**（スマホのアイコン）
2. **画面幅を変えてテスト**：
   - 1200px：デスクトップ表示
   - 768px：タブレット表示
   - 375px：スマートフォン表示
3. **各ブレークポイントでスタイルが変わるか確認**

## 🎓 解説

### ポイント1: メディアクエリの基本構文

```css
@media (max-width: 800px) {
    /* 800px以下の画面幅に適用されるスタイル */
}
```

- `@media`: メディアクエリの開始
- `max-width: 800px`: 画面幅が800px以下の場合
- `min-width`を使うと、指定した幅以上の場合に適用される

### ポイント2: ブレークポイントの順序

```css
/* デスクトップ（デフォルト） */
.container {
    max-width: 800px;
}

/* タブレット */
@media (max-width: 800px) {
    .container {
        max-width: 90%;
    }
}

/* スマートフォン */
@media (max-width: 480px) {
    .container {
        padding: 20px;
    }
}
```

- デスクトップのスタイルをデフォルトとして記述
- 画面幅が小さくなるにつれて、メディアクエリで上書き
- 順序は「大きい画面 → 小さい画面」が一般的

### ポイント3: レスポンシブな画像

```css
img {
    max-width: 100%;
    height: auto;
}
```

- `max-width: 100%`: 親要素の幅を超えない
- `height: auto`: 縦横比を維持したまま高さを自動調整
- これにより、どんな画面サイズでも画像がはみ出さない

### ポイント4: パディングとマージンの調整

```css
@media (max-width: 480px) {
    body {
        margin: 10px; /* スマホでは余白を小さく */
    }
    
    .container {
        padding: 20px; /* スマホでは内側の余白も小さく */
    }
}
```

- 小さい画面では、余白を減らして表示領域を確保
- ただし、読みやすさを損なわない程度に調整

### ポイント5: フォントサイズの調整

```css
@media (max-width: 480px) {
    h1 {
        font-size: 24px; /* スマホでは見出しを小さく */
    }
}
```

- 小さい画面では、フォントサイズを縮小
- 見出しの階層構造は維持する（h1 > h2 > h3）

## 🔍 よくある間違い

### 間違い1: メディアクエリの順序

```css
/* ❌ 間違い（スマホのスタイルがタブレットで上書きされる） */
@media (max-width: 480px) {
    h1 { font-size: 24px; }
}

@media (max-width: 800px) {
    h1 { font-size: 28px; }
}

/* ✅ 正しい（大きい画面から小さい画面の順） */
@media (max-width: 800px) {
    h1 { font-size: 28px; }
}

@media (max-width: 480px) {
    h1 { font-size: 24px; }
}
```

### 間違い2: 画像の幅指定

```css
/* ❌ 間違い（固定幅だとはみ出す） */
img {
    width: 400px;
}

/* ✅ 正しい（最大幅を100%に） */
img {
    max-width: 100%;
    height: auto;
}
```

### 間違い3: ビューポートの設定忘れ

HTMLファイルの`<head>`内に、以下のメタタグを追加する必要があります：

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```

これがないと、スマートフォンで正しく表示されません。
