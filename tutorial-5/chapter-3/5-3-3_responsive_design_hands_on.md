## 🎯 演習課題：自己紹介ページをレスポンシブ対応にしよう

### 課題の概要

これまで作成してきた自己紹介ページを、スマートフォンやタブレットでも見やすいように、レスポンシブ対応にしてください。

### 📋 要件

`style.css`にメディアクエリを追加して、以下の要件を満たしてください。

#### 1. ブレークポイントの設定

以下の3つのブレークポイントを設定します：

- **デスクトップ**：801px以上（デフォルト）
- **タブレット**：481px〜800px
- **スマートフォン**：480px以下

#### 2. タブレット（481px〜800px）のスタイル

- `.container`の最大幅を`90%`に変更
- `.container`のパディングを`30px`に変更
- `h1`のフォントサイズを`28px`に変更
- `h2`のフォントサイズを`20px`に変更

#### 3. スマートフォン（480px以下）のスタイル

- `body`のマージンを`10px`に変更
- `.container`のパディングを`20px`に変更
- `h1`のフォントサイズを`24px`に変更
- `h2`のフォントサイズを`18px`に変更
- `ul, ol`の左マージンを`10px`に変更

#### 4. 画像のレスポンシブ対応

すべてのデバイスで画像が親要素の幅に合わせて表示されるように、以下のスタイルを追加してください：

```css
img {
    max-width: 100%;
    height: auto;
}
```

## 💡 ヒント

詰まったときは、以下のヒントを参考にしてください。

### ヒント1: メディアクエリの基本構文

```css
@media (max-width: 800px) {
    /* タブレット以下のスタイル */
    .container {
        max-width: 90%;
    }
}
```

### ヒント2: ブレークポイントの順序

```css
/* デスクトップ（デフォルト） */
.container {
    max-width: 800px;
}

/* タブレット */
@media (max-width: 800px) {
    .container {
        max-width: 90%;
    }
}

/* スマートフォン */
@media (max-width: 480px) {
    .container {
        padding: 20px;
    }
}
```

### ヒント3: 複数のプロパティを変更

```css
@media (max-width: 480px) {
    h1 {
        font-size: 24px;
    }
    
    h2 {
        font-size: 18px;
    }
}
```

## ✅ 完成イメージ

完成すると、以下のようにデバイスごとに最適化されたデザインになります：

**デスクトップ（801px以上）**：
- 幅800pxの白いカード
- 大きな見出し（h1: 32px、h2: 24px）
- 余裕のあるパディング（40px）

**タブレット（481px〜800px）**：
- 幅90%のカード
- やや小さい見出し（h1: 28px、h2: 20px）
- 適度なパディング（30px）

**スマートフォン（480px以下）**：
- 幅いっぱいのカード
- 小さい見出し（h1: 24px、h2: 18px）
- コンパクトなパディング（20px）
- リストの左マージンを縮小

## 🎓 解説

### ポイント1: メディアクエリの基本構文

```css
@media (max-width: 800px) {
    /* 800px以下の画面幅に適用されるスタイル */
}
```

- `@media`: メディアクエリの開始
- `max-width: 800px`: 画面幅が800px以下の場合
- `min-width`を使うと、指定した幅以上の場合に適用される

### ポイント2: ブレークポイントの順序

```css
/* デスクトップ（デフォルト） */
.container {
    max-width: 800px;
}

/* タブレット */
@media (max-width: 800px) {
    .container {
        max-width: 90%;
    }
}

/* スマートフォン */
@media (max-width: 480px) {
    .container {
        padding: 20px;
    }
}
```

- デスクトップのスタイルをデフォルトとして記述
- 画面幅が小さくなるにつれて、メディアクエリで上書き
- 順序は「大きい画面 → 小さい画面」が一般的

### ポイント3: レスポンシブな画像

```css
img {
    max-width: 100%;
    height: auto;
}
```

- `max-width: 100%`: 親要素の幅を超えない
- `height: auto`: 縦横比を維持したまま高さを自動調整
- これにより、どんな画面サイズでも画像がはみ出さない

### ポイント4: パディングとマージンの調整

```css
@media (max-width: 480px) {
    body {
        margin: 10px; /* スマホでは余白を小さく */
    }
    
    .container {
        padding: 20px; /* スマホでは内側の余白も小さく */
    }
}
```

- 小さい画面では、余白を減らして表示領域を確保
- ただし、読みやすさを損なわない程度に調整

### ポイント5: フォントサイズの調整

```css
@media (max-width: 480px) {
    h1 {
        font-size: 24px; /* スマホでは見出しを小さく */
    }
}
```

- 小さい画面では、フォントサイズを縮小
- 見出しの階層構造は維持する（h1 > h2 > h3）

## 🔍 よくある間違い

### 間違い1: メディアクエリの順序

```css
/* ❌ 間違い（スマホのスタイルがタブレットで上書きされる） */
@media (max-width: 480px) {
    h1 { font-size: 24px; }
}

@media (max-width: 800px) {
    h1 { font-size: 28px; }
}

/* ✅ 正しい（大きい画面から小さい画面の順） */
@media (max-width: 800px) {
    h1 { font-size: 28px; }
}

@media (max-width: 480px) {
    h1 { font-size: 24px; }
}
```

### 間違い2: 画像の幅指定

```css
/* ❌ 間違い（固定幅だとはみ出す） */
img {
    width: 400px;
}

/* ✅ 正しい（最大幅を100%に） */
img {
    max-width: 100%;
    height: auto;
}
```

### 間違い3: ビューポートの設定忘れ

HTMLファイルの`<head>`内に、以下のメタタグを追加する必要があります：

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```

これがないと、スマートフォンで正しく表示されません。
