# Tutorial 5-1-2: HTMLとは何か

## 🎯 このセクションで学ぶこと

*   HTMLが、Webページの「骨格」を作るための言語であることを理解する。
*   HTMLが、プログラミング言語ではなく、「マークアップ言語」であることを理解する。
*   タグを使って、テキストに意味付け（マークアップ）をする、というHTMLの基本的な考え方を学ぶ。

---

## 導入

前のセクションで、Webページは、サーバーから送られてくる「データ」を、ブラウザが解釈して表示している、ということを学びました。

では、その「データ」の正体とは、一体何なのでしょうか？

その中心的な存在が、今回学ぶ **HTML (HyperText Markup Language)** です。HTMLは、Webページに、どのような**内容**を、どのような**構造**で表示するかを定義するための言語です。

これを、「**建物の建築**」に例えるなら、HTMLは、柱や梁、壁といった、建物の**骨格**を組み立てる作業に相当します。どんなに美しい装飾（CSS）や、便利な設備（JavaScript）があっても、しっかりとした骨格がなければ、建物は成り立ちません。HTMLは、全てのWebページの、まさに土台となる、最も重要な要素なのです。

---

## 詳細解説

### 🦴 HTML: Webページの骨格

HTMLは、**HyperText Markup Language** の略です。この長い名前を、3つの部分に分解してみましょう。

1.  **HyperText (ハイパーテキスト)**
    *   これは、単なるテキスト（文字）を超えた、「**すごいテキスト**」くらいの意味です。何がすごいのかというと、テキストの中に、**リンク (Link)** を埋め込むことで、他の文書や画像、動画など、様々な情報にジャンプできる機能を持っている点です。
    *   Webページ上の青い文字をクリックすると、別のページに移動しますよね？あれこそが、ハイパーテキストの最も代表的な機能です。この「文書から文書へ、自由に飛び回れる」という仕組みが、World Wide Web (WWW) の名前の由来にもなっています。

2.  **Markup (マークアップ)**
    *   これが、HTMLの最も本質的な部分です。「マークアップ」とは、「**印（マーク）を付ける**」という意味です。
    *   HTMLでは、文章の各部分に対して、「これは見出しですよ」「これは段落ですよ」「これはリストですよ」といったように、**タグ (Tag)** と呼ばれる、特別な印を付けていきます。
    *   この「意味の印」を付けることで、ただのテキストの羅列が、構造化された、意味のある文書に変わるのです。

3.  **Language (言語)**
    *   HTMLは、コンピュータ（ブラウザ）と、Webページ制作者が、コミュニケーションをとるための「言語」です。ただし、重要なのは、HTMLは、条件分岐や繰り返しといった、複雑な処理を行う「プログラミング言語」とは、少し違うということです。
    *   HTMLは、あくまで、文書の構造と意味を定義するための「**マークアップ言語**」に分類されます。計算などはできませんが、ブラウザに対して、「このテキストを、このように表示してね」と、的確に指示を出すことができます。

### 🏷️ タグで意味を伝える

HTMLの心臓部である「タグ」は、どのように機能するのでしょうか。

タグは、通常、`<` と `>` の記号で囲まれた、英単語や略語で表現されます。そして、多くの場合、**開始タグ**と**終了タグ**のペアで、テキストを挟み込みます。

例えば、あるテキストが「重要な見出し」であることを示したい場合、`<h1>` という開始タグと、`</h1>` という終了タグで、そのテキストを囲みます。

```html
<h1>これは重要な見出しです</h1>
```

*   `<h1>`: 「ここからレベル1の見出しが始まりますよ」という印（開始タグ）。`h` は "heading" の略。
*   `</h1>`: 「ここで見出しは終わりですよ」という印（終了タグ）。終了タグには、スラッシュ `/` が付きます。

同様に、段落を示したい場合は、`<p>` タグ（"paragraph"の略）を使います。

```html
<p>これは一つの段落です。文章が続きます。</p>
```

ブラウザは、このHTMLコードを受け取ると、タグの意味を解釈します。

*   「`<h1>` で囲まれているから、このテキストは、ページのメインタイトルだな。じゃあ、文字を大きく、太く表示しよう」
*   「`<p>` で囲まれているから、これは段落だな。じゃあ、前後に少しスペースを空けて、読みやすく表示しよう」

このように、HTMLタグは、テキストの**見た目**を直接指定するのではなく、その部分が持つ**意味**や**役割**を、ブラウザに伝えます。その結果として、ブラウザが、それぞれの意味にふさわしい、デフォルトのスタイル（見た目）を適用してくれるのです。

> **重要なポイント**: HTMLの役割は、あくまで「骨格」と「意味付け」です。文字の色や大きさ、配置といった、具体的な「デザイン」は、次のチャプターで学ぶ **CSS (Cascading Style Sheets)** という、別の言語の役割になります。役割分担を意識することが、綺麗なコードを書くための第一歩です。

## ✨ まとめ

このセクションでは、HTMLがどのような言語なのか、その本質について学びました。

*   **HTMLの役割**: Webページの**内容**と**構造**を定義する、「骨格」の部分を担当する。
*   **マークアップ言語**: プログラミング言語とは異なり、テキストの各部分に**タグ**で「印」を付け、その**意味**や**役割**を定義する言語。
*   **タグ**: `<` と `>` で囲まれたキーワードで、多くは開始タグと終了タグのペアで、テキストを挟む。（例: `<h1>見出し</h1>`, `<p>段落</p>`）
*   **HTMLとブラウザの関係**: 制作者がHTMLで意味付けを行い、ブラウザがその意味を解釈して、適切なスタイルで画面に表示する。

HTMLは、Web開発の世界への入り口です。この「タグで意味を定義する」というシンプルなルールを理解すれば、あなたはもう、Webページの構造を、自分の手で組み立て始めることができます。
