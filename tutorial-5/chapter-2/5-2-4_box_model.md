# Tutorial 5-2-4: ボックスモデル

## 🎯 このセクションで学ぶこと

*   全てのHTML要素が、「箱（ボックス）」として扱われるという、CSSの基本的な概念「ボックスモデル」を理解する。
*   ボックスを構成する4つの領域（Content, Padding, Border, Margin）の役割を理解する。
*   `width`, `height`, `padding`, `border`, `margin` といったプロパティを使って、要素のサイズや余白をコントロールする方法を学ぶ。

---

## 導入

Webページ上の、見出し、段落、画像など、全てのHTML要素は、実は、目に見えない「**箱（ボックス）**」に入っている、と考えることができます。CSSのレイアウトは、この「箱」を、どのように配置し、どのように装飾するのか、という考え方に基づいています。この、HTML要素を「箱」として捉える概念を、**ボックスモデル (Box Model)** と呼びます。

ボックスモデルを理解することは、CSSでレイアウトを組む上で、避けては通れない、最も重要なステップです。これを理解すれば、「なぜか要素が、思った通りの場所に配置されない」「要素と要素の間の、余白がうまく調整できない」といった、多くの初学者がつまずく問題の、原因がわかるようになります。

このセクションでは、ボックスという「箱」が、どのようなパーツで構成されているのかを、詳しく見ていきましょう。まるで、「**マトリョーシカ人形**」のように、入れ子構造になった箱の仕組みを、解き明かしていきます。

---

## 詳細解説

CSSのボックスモデルでは、一つの「箱」は、以下の4つの領域から構成されています。

1.  **Content (コンテンツ)**: テキストや画像など、要素の**内容**が表示される、中心の領域。
2.  **Padding (パディング)**: コンテンツと、後述するボーダー（枠線）との間の、**内側の余白**。
3.  **Border (ボーダー)**: パディングの外側にある、**枠線**。
4.  **Margin (マージン)**: ボックスの最も外側にある、**他の要素との間の余白**。

### 📦 ボックスの各領域を操るプロパティ

#### 1. コンテンツ領域: `width` と `height`

*   **役割**: コンテンツ領域の**幅 (width)** と**高さ (height)** を指定します。

```css
div {
  width: 300px; /* 幅を300ピクセルに */
  height: 200px; /* 高さを200ピクセルに */
  background-color: lightblue;
}
```

**注意点**: この `width` と `height` が指定するのは、あくまでコンテンツ領域のサイズです。実際の要素全体の大きさは、この後に出てくる、PaddingやBorderのサイズも、加算されて決まります。（この挙動を変更する `box-sizing` というプロパティもありますが、まずは基本を理解しましょう）

#### 2. パディング (内側の余白): `padding`

*   **役割**: コンテンツとボーダーの間の、内側の余白（詰め物）を指定します。`padding` を大きくすると、箱が内側から膨らむようなイメージです。
*   **書き方**: 上下左右の値を、まとめて、または個別に指定できます。

```css
div {
  /* 上下左右、全てに20pxのパディング */
  padding: 20px;

  /* 上下に10px、左右に20pxのパディング */
  padding: 10px 20px;

  /* 上、右、下、左の順に、個別に指定 */
  padding: 10px 20px 30px 40px;

  /* 特定の辺だけを指定 */
  padding-top: 10px;
  padding-right: 20px;
  padding-bottom: 30px;
  padding-left: 40px;
}
```

#### 3. ボーダー (枠線): `border`

*   **役割**: ボックスの枠線を指定します。線の**太さ**、**種類**、**色**を、まとめて指定するのが一般的です。

```css
div {
  /* 2pxの太さで、実線(solid)の、黒色(#000)の枠線 */
  border: 2px solid #000;
}
```

*   **線の種類 (style)**: `solid` (実線), `dashed` (破線), `dotted` (点線) などがあります。
*   パディングと同様に、`border-top`, `border-right` のように、辺ごとに、個別のスタイルを指定することも可能です。

#### 4. マージン (外側の余白): `margin`

*   **役割**: ボックスの**外側**の余白、つまり、**他の要素との距離**を指定します。
*   **書き方**: `padding` と同じように、上下左右の値を、まとめて、または個別に指定できます。

```css
div {
  /* 上下左右、全てに30pxのマージン */
  margin: 30px;

  /* 特定の辺だけを指定 */
  margin-top: 10px;
  margin-bottom: 20px;
}
```

*   **マージンの相殺 (Margin Collapsing)**: CSSの少しトリッキーな特徴として、**縦に隣接するボックスの `margin-bottom` と `margin-top` は、大きい方の値に「相殺」される**、というルールがあります。例えば、上の要素の `margin-bottom` が20px、下の要素の `margin-top` が30pxの場合、2つの要素間の余白は、50px (20 + 30) ではなく、30pxになります。これは、意図しない挙動に見えることがありますが、文章の段落間の余白が、自然に見えるようにするための仕様です。最初は、「そういうものか」と覚えておけば大丈夫です。

## ✨ まとめ

このセクションでは、CSSレイアウトの基礎となる「ボックスモデル」について学びました。

*   全てのHTML要素は、**Content**, **Padding**, **Border**, **Margin** の4つの領域から成る「箱」である。
*   **`width`, `height`**: コンテンツ領域のサイズを指定する。
*   **`padding`**: 箱の**内側**の余白を指定する。
*   **`border`**: 箱の**枠線**を指定する。
*   **`margin`**: 箱の**外側**の余白（他の要素との距離）を指定する。

これらのプロパティを使いこなすことで、要素の大きさや、要素間のスペースを、ピクセル単位で、正確にコントロールすることができます。Webページのレイアウトを考えるときは、常に、この「箱」を意識することが重要です。

次のセクションでは、これらの「箱」を、実際に、ページ上に、どのように並べていくのか、レイアウトの基礎となる Flexbox について学びます。
