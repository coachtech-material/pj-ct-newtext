# Tutorial 8-2-4: データの取得（SELECT）

## 🎯 このセクションで学ぶこと

*   `SELECT` 文を使って、テーブルから、データを、取得できるようになる。
*   特定の、カラムだけを、選択して、取得する、方法を、理解する。
*   `*`（アスタリスク）を、使って、全ての、カラムを、取得する、方法を、理解する。
*   `AS` を使って、カラムに、別名を、つける、方法を、知る。
*   phpMyAdminの、GUI操作が、内部的に、`SELECT` 文を、実行していることを、理解する。

---

## 導入

これまでに、データベースと、テーブルを、作成し (`CREATE`)、データを、挿入しました (`INSERT`)。しかし、データベースの、最も、重要な、役割は、蓄積した、膨大な、データの中から、**必要な、情報を、必要な時に、取り出す**ことです。そのための、主役となるのが、DQL（データ問い合わせ言語）の、**`SELECT`** 文です。

`SELECT` 文は、SQLの中で、最も、よく使われ、そして、最も、奥が深い、コマンドです。Webアプリケーションが、画面に、何かを、表示する時、その、裏側では、ほぼ、間違いなく、`SELECT` 文が、実行されています。

このセクションでは、`SELECT` 文の、最も、基本的な、使い方を、マスターし、テーブルに、格納した、情報を、自在に、取り出す、第一歩を、踏み出します。

---

## 詳細解説

### 🔍 全てのデータを取得する (`SELECT * FROM ...`)

まず、テーブルに、入っている、全ての、レコードの、全ての、カラムを、取得する、最も、シンプルな、`SELECT` 文を見てみましょう。

```sql
SELECT * FROM users;
```

*   `SELECT *`: 全ての、カラムを、選択します。`*` は、「全て」を、意味する、ワイルドカードです。
*   `FROM users`: `users` テーブルから、データを、取得します。

この、SQL文を、phpMyAdminで、実行すると、前のセクションで、`INSERT` した、ユーザーの、情報が、一覧で、表示されます。実は、phpMyAdminで、「表示」タブを、クリックした時に、内部的に、実行されているのが、この、`SELECT * FROM users;` なのです。

**[ここに、`SELECT * FROM users;` の、実行結果の、スクリーンショットを、挿入]**

### 📌 特定のカラムだけを取得する

全ての、カラムが、常に、必要とは、限りません。例えば、ユーザーの、名前と、メールアドレスの、一覧だけが、欲しい、場合も、あります。そのような時は、`*` の代わりに、取得したい、カラム名を、カンマで、区切って、指定します。

```sql
SELECT name, email FROM users;
```

この、SQL文を、実行すると、`id` や `password`, `created_at` は、含まれず、`name` と `email` の、2つの、カラムだけが、結果として、返されます。

**[ここに、`SELECT name, email FROM users;` の、実行結果の、スクリーンショットを、挿入]**

このように、必要な、データだけを、取得することは、**パフォーマンスの、観点からも、非常に、重要**です。不要な、データを、取得しないことで、データベースへの、負荷を、減らし、アプリケーションの、応答速度を、向上させることができます。

### 🏷️ カラムに別名をつける (`AS`)

取得した、カラム名が、そのままだと、分かりにくい場合や、他の、テーブルの、カラムと、名前が、重複してしまう、場合に、`AS` を使って、別名を、つけることができます。

```sql
SELECT
    name AS user_name,
    email AS user_email,
    created_at AS registration_date
FROM
    users;
```

*   `name AS user_name`: `name` カラムを、`user_name` という、名前で、取得します。
*   `email AS user_email`: `email` カラムを、`user_email` という、名前で、取得します。
*   `created_at AS registration_date`: `created_at` カラムを、`registration_date` という、名前で、取得します。

この、SQL文を、実行すると、結果の、ヘッダー部分が、`name` や `email` ではなく、`user_name` や `user_email` に、変わっていることが、確認できます。

**[ここに、`AS` を使った、SELECT文の、実行結果の、スクリーンショットを、挿入]**

`AS` は、省略することも可能ですが、可読性のために、明記することが、推奨されます。

```sql
-- ASを省略した書き方
SELECT name user_name, email user_email FROM users;
```

### ⚙️ phpMyAdminでの操作との関連

phpMyAdminの、GUI操作と、`SELECT` 文の、関係を、改めて、整理しておきましょう。

*   **「表示」タブをクリック**: `SELECT * FROM table_name;` が、実行される。
*   **テーブル名をクリック**: `SELECT * FROM table_name;` が、実行される。
*   **カラムのヘッダーをクリックしてソート**: `SELECT * FROM table_name ORDER BY column_name;` のような、SQLが、実行される。（`ORDER BY` は、後のセクションで学びます）

このように、GUIでの、便利な、操作も、裏側では、SQL文が、組み立てられ、実行されています。このことを、意識するだけで、phpMyAdminが、単なる、ツールから、強力な、SQL学習支援ツールへと、変わります。

## ✨ まとめ

このセクションでは、データベースから、データを、取り出すための、`SELECT` 文の、基本を学びました。

*   テーブルから、データを、取得するには、`SELECT ... FROM ...` 文を、使う。
*   全ての、カラムを、取得するには、`*`（アスタリスク）を、使う。
*   特定の、カラムだけを、取得するには、カラム名を、カンマで、区切って、指定する。これは、パフォーマンス上、重要である。
*   `AS` を使うことで、取得する、カラムに、分かりやすい、別名を、つけることができる。
*   phpMyAdminの、データ表示機能は、内部的に、`SELECT` 文を、実行している。

`SELECT` 文は、これから、学ぶ、`WHERE`（条件指定）、`JOIN`（テーブル結合）、`GROUP BY`（集計）といった、句と、組み合わせることで、その、真価を、発揮します。
