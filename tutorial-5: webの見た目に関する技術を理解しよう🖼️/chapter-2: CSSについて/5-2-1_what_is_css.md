# Tutorial 5-2-1: CSSとは何か

## 📌 BEエンジニア志望者へ

このChapterで学ぶCSSは、BEエンジニアとして**すぐに実装できるようになる必要はありません**。ただし、以下の理由から、概念として理解しておくことが重要です。

- フロントエンドエンジニアとの協業時に、共通言語として話せるようになる
- 提供されたテンプレートのコードを読んで理解できるようになる
- 簡単な修正（文言変更、色の微調整など）ができるようになる

「こういうものがある」「こういう仕組みで動いている」という概念を押さえることを目標にしてください。

## 導入

前のチャプターで、あなたはHTMLを使って、Webページの「骨格」を組み立てる方法を学びました。しかし、HTMLだけで作られたWebページは、まるで、骨格がむき出しの建物のようです。構造はしっかりしていますが、無機質で、魅力的とは言えません。

そこで登場するのが、**CSS (Cascading Style Sheets)** です。CSSは、Webページの**見た目（スタイル）**を専門に担当する言語です。文字の色や大きさ、背景色、レイアウトなど、デザインに関することを、CSSでコントロールします。

「建物の建築」の例えを続けるなら、CSSは、壁紙を貼ったり、ペンキを塗ったり、家具を配置したりといった、**内装や外装のデザイン**を担当する作業です。HTMLという骨格に、CSSという装飾を施すことで、Webページは、初めて、美しく、使いやすいものになるのです。

---

## 詳細解説

### 🎨 CSS: Webページのデザイナー

CSSは、**Cascading Style Sheets** の略です。

*   **Cascading (カスケーディング)**: 「滝のように流れ落ちる」という意味です。複数のスタイルが競合した場合に、どのスタイルを優先するかを決めるルールがあります。
*   **Style (スタイル)**: 文字通り「見た目」のことです。色、サイズ、フォント、余白、配置などを指します。
*   **Sheets (シーツ)**: CSSは通常、HTMLファイルとは別の、専用のファイル（`.css` ファイル）に記述されます。

### 🤝 HTMLとCSSの役割分担

Web開発の最も重要な原則の一つが、「**構造と見た目の分離**」です。

*   **HTML**: 文書の**構造**と**意味**に責任を持つ。（「これは見出しである」「これはリストである」）
*   **CSS**: 文書の**見た目（プレゼンテーション）**に責任を持つ。（「見出しは青色で大きな文字にする」）

#### なぜ役割分担が重要なのか？

**メンテナンス性の向上**が最大の理由です。

```css
/* style.css */
h1 {
  color: blue;
  font-size: 32px;
}
```

CSSを使えば、スタイルを一箇所にまとめることができます。色の変更が必要になったら、このCSSファイルを書き換える**だけ**で、サイト全体の全ての`<h1>`タグの見た目が、一瞬で変わります。

---

## ✨ まとめ

*   **CSSの役割**: Webページの**見た目（デザイン）**を専門に担当する言語。
*   **HTMLとの関係**: HTMLが「骨格」なら、CSSは「装飾」。HTMLで構造を定義し、CSSで見た目を整える。
*   **構造と見た目の分離**: この原則を守ることで、メンテナンス性が向上する。
