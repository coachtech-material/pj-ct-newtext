# Tutorial 3-8: ショートカットキーで効率化

## 🎯 このセクションで学ぶこと

*   コマンドラインでの入力を劇的に効率化する、基本的なショートカットキーを学ぶ。
*   過去に実行したコマンドを簡単に呼び出して、再利用する方法を学ぶ。
*   入力中のコマンドを、素早く編集する方法を学ぶ。

---

## 導入

コマンドライン操作に慣れてくると、同じようなコマンドを何度も入力したり、長いファイルパスを打ち込んだりする場面が増えてきます。そんな時、一文字ずつキーボードを叩いていては、時間がかかって仕方ありません。マウスを使わずにキーボードだけで操作するのがコマンドラインの魅力ですが、その効率は、ショートカットキーを知っているかどうかに大きく左右されます。

プロのエンジニアが、まるで魔法のように高速でコマンドを打ち込んでいるように見えるのは、彼らがタイピングが速いから、というだけではありません。彼らは、これから紹介するようなショートカットキーを駆使して、「**いかにキーを打つ回数を減らすか**」を常に考えているのです。

このセクションで学ぶショートカットキーは、一度覚えてしまえば、あなたのコマンドライン操作のスピードと快適さを、劇的に向上させてくれます。最初は意識して使う必要がありますが、すぐに指が覚えて、無意識に使えるようになります。これは、自転車の乗り方を覚えるのに似ています。一度身につければ、一生もののスキルになります。

---

## 詳細解説

### ⚙️ 1. 補完機能: `Tab` キー

**`Tab` キーによる補完は、コマンドラインで最も重要で、最も頻繁に使う機能です。これを知らないと、コマンドラインを使っている意味が半減すると言っても過言ではありません。**

`Tab` キーは、入力中のコマンド名、ファイル名、ディレクトリ名を、文脈から推測して**自動で補完**してくれます。

#### ファイル名・ディレクトリ名の補完

`~/practice` ディレクトリに移動し、`ls` で `dir1_copy` というディレクトリがあることを確認してください。

ここで、`cd dir` まで入力して、`Tab` キーを押してみてください。

```bash
cd dir
# (Tabキーを押す)
```

`dir` で始まるものが `dir1` と `dir1_copy` の2つあるため、途中までしか補完されません。もう一度 `Tab` キーを押すと、候補の一覧が表示されます。

ここで、`1` を追加入力して `cd dir1` とし、もう一度 `Tab` キーを押すと、今度は `_copy` まで補完されるはずです。

このように、数文字入力して `Tab` キーを押す、という操作を繰り返すことで、長いファイル名やディレクトリ名を、タイプミスなく、素早く入力できます。

#### コマンド名の補完

`Tab` キーは、コマンド名の補完にも使えます。例えば、`mkd` と入力して `Tab` キーを押せば、`mkdir` に補完されます。

### ⚙️ 2. 履歴の活用: `↑` `↓` キーと `Ctrl + R`

#### `↑` (上矢印) / `↓` (下矢印) キー

`↑` キーを押すと、一つ前に実行したコマンドが呼び出されます。繰り返し押すことで、どんどん過去のコマンドに遡ることができます。`↓` キーはその逆です。

同じコマンドを少しだけ修正して、もう一度実行したい場合に非常に便利です。

#### `Ctrl + R`: 履歴の検索 (Read-line)

`↑` キーを何度も押して過去のコマンドを探すのは大変です。そんな時は、`Ctrl + R` を使います。

`Ctrl + R` を押すと、プロンプトが `(reverse-i-search)` という表示に変わります。ここで、探したいコマンドに含まれるキーワード（例えば `mkdir`）を入力し始めると、履歴の中からそのキーワードを含むコマンドが、リアルタイムで検索・表示されます。

目的のコマンドが見つかったら、`Enter` キーを押せば、そのコマンドが実行されます。矢印キー（`→` や `←`）を押せば、実行する前にコマンドを編集することもできます。

### ⚙️ 3. カーソル移動と編集

入力中の長いコマンドの、先頭の方にタイプミスを見つけたとします。`←` キーを何度も押してカーソルを移動するのは面倒ですね。そんな時に役立つのが、カーソル移動のショートカットです。

| ショートカット | 説明 |
| :--- | :--- |
| `Ctrl + A` | 行の**先頭**にカーソルを移動 (Ahead) |
| `Ctrl + E` | 行の**末尾**にカーソルを移動 (End) |
| `Ctrl + B` | 1文字**後ろ**に移動 (Backward) ( `←` と同じ) |
| `Ctrl + F` | 1文字**前**に移動 (Forward) ( `→` と同じ) |

`Ctrl + A` と `Ctrl + E` は、特に使用頻度が高いので、必ず覚えましょう。

### ⚙️ 4. 削除とクリア

| ショートカット | 説明 |
| :--- | :--- |
| `Ctrl + U` | カーソル位置から**行頭まで**を削除 |
| `Ctrl + K` | カーソル位置から**行末まで**を削除 |
| `Ctrl + W` | カーソル位置から**前の単語の先頭まで**を削除 |
| `Ctrl + L` | 画面を**クリア** (`clear` コマンドと同じ) |

入力したコマンドがごちゃごちゃになって、最初からやり直したい時は、`Ctrl + U` で一気に消去できます。また、ターミナルの表示が乱れたり、過去の出力が邪魔になったりした時は、`Ctrl + L` で画面をリフレッシュするとスッキリします。

## ✨ まとめ

このセクションでは、コマンドライン操作を高速化するための、必須のショートカットキーを学びました。これらは、最初は意識して使わないと身につきませんが、慣れればあなたの生産性を飛躍的に向上させてくれます。

**【最重要】**

| キー | 機能 |
| :--- | :--- |
| `Tab` | **補完**。コマンド名、ファイル名、ディレクトリ名を自動入力。 |
| `↑` / `↓` | **履歴**。過去に実行したコマンドを呼び出す。 |
| `Ctrl + R` | **履歴検索**。キーワードで過去のコマンドを検索する。 |

**【覚えておくと超便利】**

| キー | 機能 |
| :--- | :--- |
| `Ctrl + A` | 行頭に移動 |
| `Ctrl + E` | 行末に移動 |
| `Ctrl + L` | 画面をクリア |
| `Ctrl + U` | 行頭まで削除 |

特に `Tab` キーは、もはやショートカットというより、コマンドラインの基本操作の一部です。ファイルやディレクトリを指定する際は、必ず `Tab` キーで補完する癖をつけましょう。タイプミスを防ぎ、時間を節約するという、二重のメリットがあります。
