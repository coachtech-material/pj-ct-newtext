# Tutorial 3-2: ターミナルの起動と基本操作

## 🎯 このセクションで学ぶこと

*   VSCodeに内蔵されている「統合ターミナル」を起動し、操作できるようになる。
*   プロンプトの基本的な見方を理解する。
*   最も基本的な2つのコマンド `pwd` と `ls` を実行し、その意味を理解する。

---

## 導入

前のセクションで、コマンドラインの概念について学びました。いよいよ、コマンドを入力するための「窓口」である**ターミナル**を実際に起動し、最初のコマンドを打ち込んでみましょう。

このコースでは、OS標準のターミナル（Windows TerminalやMacのターミナル.app）ではなく、**VSCodeに内蔵されている「統合ターミナル**」を一貫して使用します。なぜなら、コードを書くエディタ画面と、コマンドを打つターミナル画面が同じウィンドウ内に表示されるため、視線の移動が少なく、開発に集中できるからです。

「黒い画面に、最初の文字を打ち込む…」

少し緊張するかもしれませんが、大丈夫です。このセクションで実行するのは、コンピュータに何かを変更させるような危険なコマンドではありません。ただ、「今どこにいるか教えて？」「周りには何があるの？」と、コンピュータに質問するだけの、安全なコマンドです。

さあ、シェフ（シェル）との対話の第一歩を踏み出しましょう。

---

## 詳細解説

### ⚙️ 1. VSCodeで統合ターミナルを起動する

まず、VSCodeを起動し、Tutorial 2で作成した `hello-world` フォルダを開いてください。（「ファイル」>「最近使用したものを開く」から選択すると便利です）

VSCodeでターミナルを起動する方法は、いくつかあります。

*   **メニューから開く**: 上部のメニューバーから「ターミナル」>「新しいターミナル」を選択します。
*   **ショートカットキーで開く**: `Ctrl + @` （Windows/Mac共通）を押します。

どちらかの方法で、VSCodeの画面下部に、ターミナル（パネル）が表示されれば成功です。

（ここにスクリーンショットを挿入: VSCodeの下部にターミナルが表示された状態。メニューとショートカットキーを指し示す）

### ⚙️ 2. プロンプトを理解する

ターミナルが起動すると、以下のような文字列が表示され、その右側でカーソルが点滅しているはずです。この、コマンドの入力を促す表示のことを**プロンプト (Prompt)** と呼びます。

```bash
C:\Users\YourName\Desktop\hello-world> 
```
*(Windowsの場合の例)*

```bash
YourName@MacBook-Air hello-world % 
```
*(Macの場合の例)*

このプロンプトには、いくつか重要な情報が含まれています。

*   **`hello-world`**: これは、**カレントディレクトリ（現在地）**を示しています。つまり、「今、あなたは `hello-world` というフォルダ（ディレクトリ）の中にいますよ」ということを教えてくれています。
*   **`>` や `%`**: この記号の右側に、コマンドを打ち込んでいきます。シェルの種類によって、`$` や `#` など、表示される記号は異なります。

プロンプトは、「（現在地）にいます。ご用件（コマンド）をどうぞ」と、コンピュータがあなたに話しかけているサインなのです。

### ⚙️ 3. 最初のコマンド: `pwd` (Print Working Directory)

それでは、記念すべき最初のコマンドを打ち込んでみましょう。プロンプトの右側に、`pwd` と入力し、**Enterキー**を押してください。（コマンドは、必ず**半角の小文字**で入力します）

`pwd` は **P**rint **W**orking **D**irectory の略で、「今いる作業ディレクトリ（現在地）の、正式な住所（パス）を教えてください」というコマンドです。

```bash
# Macでの実行例
YourName@MacBook-Air hello-world % pwd
/Users/YourName/Desktop/hello-world
```

Enterキーを押すと、次の行に、今いる `hello-world` フォルダの、コンピュータの根っこからの完全な住所が表示されます。このように、コマンドラインでは、「**コマンドを入力してEnter」→「実行結果が表示される**」という流れを繰り返していきます。

### ⚙️ 4. 2つ目のコマンド: `ls` (List)

次に、「今いる場所には、どんなファイルやフォルダがありますか？」と尋ねるコマンド、`ls` を実行してみましょう。`ls` は **L**i**s**t の略です。

プロンプトの右側に `ls` と入力し、Enterキーを押してください。

```bash
# Macでの実行例
YourName@MacBook-Air hello-world % ls
index.html
```

実行結果として、`hello-world` フォルダの中にある `index.html` というファイル名が表示されました。これは、GUIでフォルダをダブルクリックして、中のファイル一覧を眺めるのと同じ操作を、コマンドで行ったことになります。

| GUIでの操作 | CLIでの操作 |
| :--- | :--- |
| `hello-world` フォルダをダブルクリックして開く | `ls` コマンドを実行する |

#### Windowsユーザー向けの補足: `dir` コマンド

Windowsの伝統的なコマンドプロンプトでは、`ls` の代わりに `dir` というコマンドが使われてきました。VSCodeのターミナルでは、`ls` も `dir` も両方使えることが多いですが、もし `ls` でエラーが出る場合は、`dir` を試してみてください。機能はほぼ同じです。

### ⚙️ 5. コマンドのクリア: `clear`

コマンドを実行していくと、画面がごちゃごちゃしてきます。そんな時は、`clear` コマンドを実行すると、ターミナルの表示を一旦リセットし、綺麗な状態にすることができます。

```bash
YourName@MacBook-Air hello-world % clear
```

Enterを押すと、過去の表示が消え、プロンプトだけが一番上に表示されます。（過去のコマンドが削除されたわけではなく、単に見えなくなっただけです）

## ✨ まとめ

このセクションでは、VSCodeの統合ターミナルを起動し、最初のコマンドを実行するまでの一連の流れを体験しました。

*   **ターミナルの起動**: VSCodeのメニューまたはショートカットキー (`Ctrl + @`) で、画面下部にターミナルを起動した。
*   **プロンプト**: コマンドの入力を促す表示であり、現在地などの情報が含まれていることを学んだ。
*   **`pwd` コマンド**: **P**rint **W**orking **D**irectory。今いる場所の「正式な住所」を表示するコマンド。
*   **`ls` コマンド**: **L**i**s**t。今いる場所にあるファイルやフォルダの「一覧」を表示するコマンド。
*   **基本サイクル**: 「コマンドを入力 → Enterキーを押す → 実行結果を確認する」という基本操作に慣れた。

たった2つのコマンドですが、これはコマンドラインの世界への、非常に大きな一歩です。GUIで何気なく行っていた「場所の確認」や「中身の確認」が、コマンドという「言葉」で実行できることを実感できたのではないでしょうか。
