# Tutorial 3-5: パスの概念（絶対パスと相対パス）

## 🎯 このセクションで学ぶこと

*   ファイルやディレクトリの「住所」である「パス」の概念を正確に理解する。
*   「絶対パス」と「相対パス」の2種類の住所表記の違いを理解する。
*   どのような場面で、どちらのパスを使えば効率的かを判断できるようになる。

---

## 導入

これまでのセクションで、`pwd` や `cd`、`mv` などのコマンドを使って、ディレクトリの場所を指定してきました。この、ファイルやディレクトリの場所を示す「住所」のことを、プログラミングの世界では**パス (Path)** と呼びます。

コマンドラインを使いこなす上で、この「パス」の概念を正確に理解することは、避けては通れない、非常に重要なステップです。パスの指定を間違えることは、コマンドラインで最もよくあるエラーの原因の一つです。

パスには、大きく分けて2種類の書き方があります。

1.  **絶対パス (Absolute Path)**: 住所を、最初から（地球から）全て書く方法。
2.  **相対パス (Relative Path)**: 「今いる場所から見て、あっち」というように、現在地を基準に書く方法。

現実世界で、友人の家に行く場面を想像してみてください。

*   **絶対パス**: 「東京都渋谷区…」のように、住所を完全に指定する方法。誰がどこから聞いても、必ず同じ場所を指します。
*   **相対パス**: 「今いるコンビニから、道を渡って、2軒隣の赤い屋根の家だよ」のように、現在地からの道順で説明する方法。話している場所が変わると、指し示す家も変わってしまいます。

どちらの指定方法にも、メリットとデメリットがあります。このセクションでは、2つのパスの違いを学び、状況に応じて適切に使い分けるスキルを身につけます。

---

## 概念の説明

### 🔑 絶対パス (Absolute Path)

絶対パスは、ファイルシステムの**ルートディレクトリ (`/`) から**始まる、完全なパスです。`pwd` コマンドで表示されるのが、この絶対パスです。

*   **特徴**: 
    *   必ず `/` (Mac/Linux) またはドライブレター (`C:\`) (Windows) から始まる。
    *   カレントディレクトリ（現在地）がどこであっても、常に同じ場所を指し示す。
    *   長くなりがちだが、間違いが起こりにくい。

*   **例 (Mac/Linux)**:
    *   `/Users/YourName/Desktop/hello-world/index.html`
    *   `/etc/hosts`

*   **使い所**: 
    *   プログラムの設定ファイルなどで、場所を固定して指定したい時。
    *   現在地から遠く離れた場所にあるファイルを、一発で指定したい時。

### 🔑 相対パス (Relative Path)

相対パスは、**カレントディレクトリ（現在地）を基準**として、目的のファイルやディレクトリまでの道順を示します。ルートディレクトリ (`/`) からは始まりません。

*   **特徴**: 
    *   `/` からは始まらない。
    *   `.` (カレントディレクトリ) や `..` (親ディレクトリ) といった特殊な記号を多用する。
    *   カレントディレクトリが変わると、指し示す場所も変わってしまう。
    *   短く記述できることが多いが、現在地を意識する必要がある。

*   **例 (カレントディレクトリが `/Users/YourName/Desktop` の場合)**:
    *   `hello-world/index.html` (今いる場所の中の、`hello-world` フォルダの中の `index.html`)
    *   `./hello-world/index.html` (`.` はカレントディレクトリなので、上と同じ意味)
    *   `../Documents/report.pdf` (一つ上の階層 (`/Users/YourName`) に戻り、そこにある `Documents` フォルダの中の `report.pdf`)

*   **使い所**: 
    *   同じプロジェクト内の、近くにあるファイルを指定する時。
    *   `cd` コマンドで、少しだけ場所を移動する時。

| 種類 | 書き始め | 基準 | 特徴 |
| :--- | :--- | :--- | :--- |
| **絶対パス** | `/` または `C:\` | ルートディレクトリ | 常に同じ場所を指す、長くなりがち |
| **相対パス** | 上記以外 | カレントディレクトリ | 現在地によって変わる、短く書ける |

## 詳細解説

それでは、`cd` コマンドを使って、絶対パスと相対パスの動きの違いを実際に体験してみましょう。

まず、練習の準備として、ホームディレクトリ (`~`) の直下に、以下のような構造のディレクトリを作成します。

```bash
# ホームに移動
cd ~

# フォルダを一気に作成
mkdir -p project/src/css
mkdir -p project/dist

# 確認
ls project
# dist src
ls project/src
# css
```

### ⚙️ 1. 相対パスでの移動

現在地を `~/project/src` にして、そこから `css` ディレクトリに移動してみましょう。

```bash
# 現在地を移動
cd ~/project/src

pwd
# /Users/YourName/project/src

# 相対パスで css に移動
cd css

pwd
# /Users/YourName/project/src/css
```

`cd css` という相対パスでの指定は、「今いる場所 (`src`) の中にある `css` に移動して」という意味になります。

次に、今いる `css` から、一つ上の `src` に戻り、そこから `dist` に移動してみましょう。

```bash
# 現在地は css

# 一つ上に戻り (src)、そこから dist に移動
cd ../dist

pwd
# /Users/YourName/project/dist
```

`../dist` という相対パスは、「一つ上の階層に戻って、そこにある `dist` フォルダに入って」という、連続した指示になります。

### ⚙️ 2. 絶対パスでの移動

今度は、絶対パスを使ってみます。現在地がどこであっても、一発で目的の場所に移動できるのが絶対パスの強みです。

```bash
# 現在地は dist

# 絶対パスで css ディレクトリに一気に移動
cd /Users/YourName/project/src/css

pwd
# /Users/YourName/project/src/css
```

`cd` の後に、`pwd` で表示されるような完全なパスを指定することで、現在地に関係なく、直接目的地にジャンプできました。

### ⚙️ 3. ホームディレクトリ (`~`) の活用

絶対パスは便利ですが、毎回 `/Users/YourName` のように長いパスを打つのは大変です。そこで、ホームディレクトリを表す `~` が役立ちます。`~` は、シェルの機能によって、自動的に `/Users/YourName` のようなあなたのホームディレクトリの絶対パスに置き換えられます。

```bash
# 現在地は css

# ~ を使って dist ディレクトリに移動
cd ~/project/dist

pwd
# /Users/YourName/project/dist
```

`cd /Users/YourName/project/dist` と書くよりも、`cd ~/project/dist` と書く方が、ずっと短くて済みますね。**自分のプロジェクト内のファイルを指定する際は、`~` から始まるパスを使うのが非常に効率的です。**

## ✨ まとめ

このセクションでは、コマンドライン操作の要である「パス」の概念について、深く学びました。

*   **パスとは**: ファイルやディレクトリの場所を示す「住所」のこと。
*   **絶対パス**: 
    *   ルート (`/`) から始まる完全な住所。
    *   `pwd` で表示されるパス。
    *   いつでも、どこからでも、同じ場所を指す。
*   **相対パス**: 
    *   現在地 (`.`) を基準にした道順。
    *   `..` (一つ上) を使って階層を移動できる。
    *   近くのファイルを指定するのに便利。
*   **`~` (チルダ)**: ホームディレクトリを指す便利なショートカット。絶対パスのように使える。

最初は、絶対パスと相対パスのどちらを使うべきか迷うかもしれません。基本的には、以下のルールで使い分けると良いでしょう。

*   `cd` で少しだけ移動する → **相対パス** (`cd ..`, `cd css`)
*   自分のプロジェクト内の、少し離れた場所に移動・指定する → **`~` を使ったパス** (`cd ~/project/dist`)
*   全く関係ない、システムの設定ファイルなどを指定する → **絶対パス** (`/etc/hosts`)
