# Tutorial 4-3-1: リポジトリの作成 (git init)

## 🎯 このセクションで学ぶこと

*   Gitリポジトリの役割を理解する。
*   `git init` コマンドを使って、新しいGitリポジトリを作成する方法を学ぶ。
*   `git init` を実行すると、`.git` という隠しディレクトリが作成されることを理解する。

---

## 導入

これまでの準備を経て、いよいよ、あなたの最初のプロジェクトで、Gitによるバージョン管理を開始します。

Gitの世界では、バージョン管理の対象となるプロジェクトのフォルダのことを、**リポジトリ (Repository)** と呼びます。「保管庫」や「貯蔵庫」といった意味の英単語です。このリポジトリという「箱」の中に、あなたのプロジェクトの全てのファイルと、その変更履歴が、安全に保管されていきます。

このセクションでは、`git init` というコマンドを使って、空っぽのディレクトリを、Gitが管理する神聖な「リポジトリ」へと変える、最初の魔法の呪文を唱えます。これは、新しいプロジェクトを開始する際に、**一番最初に行う、ただ一度きりの儀式**です。

---

## 詳細解説

### 📂 プロジェクトディレクトリの準備

まず、バージョン管理を行いたい、新しいプロジェクト用のディレクトリを作成しましょう。ここでは、`my-first-git` という名前のディレクトリを作成します。

ターミナルを開き、`cd` コマンドで、あなたがプロジェクトを管理したい場所（例えば、デスクトップや、専用の`projects`ディレクトリなど）に移動してから、以下のコマンドを実行してください。

```bash
# "my-first-git" という名前のディレクトリを作成
mkdir my-first-git

# 作成したディレクトリに移動
cd my-first-git
```

`pwd` コマンドを実行して、現在地が `my-first-git` ディレクトリの中であることを確認してください。

### ✨ `git init` の実行

現在、`my-first-git` は、ただの空っぽのディレクトリです。ここに、Gitの魂を吹き込み、リポジトリへと昇格させます。

以下のコマンドを実行してください。

```bash
git init
```

実行すると、以下のようなメッセージが表示されます。（Gitのバージョンや言語設定によって、メッセージは多少異なります）

```
Initialized empty Git repository in /Users/your-name/Desktop/my-first-git/.git/
```

「この場所に、空のGitリポジトリを初期化しました」というメッセージです。たったこれだけです。これで、`my-first-git` ディレクトリは、Gitによって監視・管理される、正式な**リポジトリ**となりました。

### 🕵️ `.git` ディレクトリの正体

`git init` を実行すると、具体的に何が起こったのでしょうか？

実は、`my-first-git` ディレクトリの中に、`.git` という名前の、**隠しディレクトリ**が一つ作成されています。確認してみましょう。

`ls` コマンドに `-a` オプションを付けて実行すると、隠しファイルや隠しディレクトリを含む、全ての項目が表示されます。

```bash
ls -a
```

実行結果に、`.` と `..` に加えて、`.git` というディレクトリが表示されているはずです。

```
. .. .git
```

この `.git` ディレクトリこそが、**Gitリポジトリの本体**です。この中に、あなたのプロジェクトの全ての変更履歴、設定情報、コミットの記録などが、Git独自の形式で保存されていきます。いわば、プロジェクトの「タイムマシン」の心臓部です。

**警告**: この `.git` ディレクトリを、**手動で編集したり、削除したりしてはいけません。** もし、このディレクトリを削除してしまうと、あなたのプロジェクトの全ての変更履歴が、永遠に失われてしまいます。このディレクトリは、Gitコマンドを通してのみ、操作するようにしてください。

---

## ✨ まとめ

このセクションでは、新しいGitリポジトリを作成する、`git init` コマンドについて学びました。

*   **リポジトリとは**: Gitでバージョン管理されるプロジェクトのフォルダのこと。
*   **`git init`**: カレントディレクトリを、Gitリポジトリとして初期化するコマンド。新しいプロジェクトの開始時に、一度だけ実行する。
*   **`.git` ディレクトリ**: `git init` によって作成される隠しディレクトリで、リポジトリの全ての情報が格納されている本体。**絶対に手動で変更・削除してはならない。**

これで、あなたのプロジェクトを管理するための「保管庫（リポジトリ）」が用意できました。しかし、今はまだ、保管庫は空っぽの状態です。

次のセクションでは、このリポジトリに、最初のファイルを追加し、バージョン管理の対象としてGitに認識させる方法を学びます。
