# Tutorial 4-4-1: リモートリポジトリの作成

## 🎯 このセクションで学ぶこと

*   リモートリポジトリの役割を再確認する。
*   GitHubのWebサイト上で、新しいリモートリポジトリを作成する方法を学ぶ。
*   PublicリポジトリとPrivateリポジトリの違いを理解する。

---

## 導入

前のチャプターまでで、あなたはGitの基本的なローカルでの操作方法をマスターしました。`init`でリポジトリを作り、`add`で変更をステージングし、`commit`で歴史を記録する。このサイクルを、自分のコンピュータの中で、完結して行えるようになりました。

しかし、Gitの真の力は、チームでの共同作業でこそ発揮されます。あなたのローカルリポジトリにある素晴らしい変更履歴を、チームメイトと共有し、一つの大きなプロジェクトを協力して作り上げていく。そのための「共有の場所」となるのが、**リモートリポジトリ**です。

このセクションでは、GitHubのWebサイト上で、あなたのローカルリポジトリの「受け皿」となる、新しいリモートリポジトリを作成します。これは、あなたのプロジェクトを、世界に向けて（あるいは、チーム内限定で）公開するための、最初のステップです。

---

## 詳細解説

### ➕ 新しいリポジトリの作成

1.  **GitHubにログイン**: まず、あなたのGitHubアカウントにログインします。

2.  **作成ページへ移動**: ログイン後のダッシュボード画面の右上にある「＋」アイコンをクリックし、ドロップダウンメニューから「New repository」を選択します。

    <!-- ここにスクリーンショットを挿入: GitHubダッシュボード右上の「＋」アイコンと「New repository」メニュー -->

    あるいは、[https://github.com/new](https://github.com/new) に直接アクセスしても構いません。

### 📝 リポジトリ情報の設定

「Create a new repository」というページが表示されます。ここで、あなたの新しいリモートリポジトリに関する、いくつかの重要な情報を設定します。

1.  **Owner (所有者)**: あなたのユーザー名が選択されていることを確認します。（将来、Organization（組織）に所属すると、ここに選択肢が増えます）

2.  **Repository name (リポジトリ名)**: 
    *   リモートリポジトリの名前を入力します。
    *   **重要**: ローカルリポジトリのディレクトリ名と、**同じ名前**にすることを、強く推奨します。今回の場合は、`my-first-git` と入力します。
    *   これにより、ローカルとリモートの関係が、直感的に分かりやすくなります。

3.  **Description (説明)** (任意):
    *   このリポジトリが、どのようなプロジェクトなのかを説明する、短い文章を入力します。他の人が、このリポジトリの目的を素早く理解するのに役立ちます。
    *   例: `My first project to learn Git and GitHub.`

4.  **Public / Private の選択**: 
    *   これは、非常に重要な選択です。このリポジトリを、誰に公開するかを設定します。
    *   **Public (公開)**: インターネット上の、**誰でも**、このリポジトリのコードを閲覧できます。オープンソースプロジェクトや、ポートフォリオとして公開したい学習プロジェクトに適しています。
    *   **Private (非公開)**: あなたが明示的に招待した、**特定の共同作業者のみ**が、このリポジトリを閲覧・操作できます。会社の業務プロジェクトや、まだ公開したくない個人プロジェクトに適しています。
    *   **今回は、学習目的ですので、どちらを選んでも構いません。** 分からなければ、`Public` を選んでおきましょう。（後から変更することも可能です）

5.  **Initialize this repository with:** (リポジトリの初期化オプション)
    *   **警告**: このセクションは、**非常に重要**です。今回は、すでにローカルにリポジトリが存在し、それをGitHubにアップロードしようとしています。そのため、GitHub上で、余計なファイルを自動生成させないように、**このセクションのチェックボックスは、全て空のままにしてください。**
    *   `Add a README file`: READMEファイルを自動生成します。→ **チェックしない**
    *   `Add .gitignore`: 特定のファイルをバージョン管理から除外するための設定ファイルを自動生成します。→ **チェックしない**
    *   `Choose a license`: ライセンスファイルを自動生成します。→ **チェックしない**

    もし、ここでファイルを追加してしまうと、ローカルリポジトリの歴史と、リモートリポジトリの歴史が、食い違ってしまい、最初の`push`が非常に面倒になります。

### ✅ リポジトリの作成

全ての設定が完了したら、ページの下部にある緑色の「Create repository」ボタンをクリックします。

おめでとうございます！これで、GitHub上に、あなたの最初の**リモートリポジトリ**が作成されました。

作成が完了すると、`Quick setup — if you’ve done this kind of thing before` というタイトルのページに移動します。このページには、今作成したばかりの空のリモートリポジトリと、あなたのローカルリポジトリを、どのように接続すればよいか、という手順が、親切に書かれています。

特に、`…or push an existing repository from the command line` というセクションに表示されているコマンドが、次のステップで重要になります。このページのURLは、あなたのリモートリポジトリの「住所」になりますので、ブックマークしておくか、タブを開いたままにしておきましょう。

## ✨ まとめ

このセクションでは、GitHub上で、新しいリモートリポジトリを作成する方法を学びました。

*   **作成方法**: GitHubにログインし、「New repository」ページで必要な情報を入力する。
*   **重要な設定項目**: 
    *   **Repository name**: ローカルのプロジェクト名と合わせるのが推奨される。
    *   **Public / Private**: リポジトリの公開範囲を選択する。学習中はどちらでも良い。
    *   **初期化オプション**: **今回は、全てのチェックを外すこと。**（既存のローカルリポジトリを`push`するため）
*   **作成後のページ**: ローカルリポジトリと接続するための、重要なコマンドが記載されている。

これで、ローカルの変更をアップロードするための「受け皿」が、クラウド上に用意できました。次のセクションでは、いよいよ、あなたのローカルリポジトリと、このリモートリポジトリを、運命の赤い糸で結びつけます。
