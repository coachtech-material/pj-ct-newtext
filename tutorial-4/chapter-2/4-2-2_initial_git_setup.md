# Tutorial 4-2-2: Gitの初期設定

## 🎯 このセクションで学ぶこと

*   Gitに、あなたの「名前」と「メールアドレス」を設定する必要性を理解する。
*   `git config` コマンドを使って、ユーザー名とメールアドレスを設定する方法を学ぶ。
*   設定した内容が、正しく保存されていることを確認する方法を学ぶ。

---

## 導入

前のセクションで、あなたは無事にGitをインストールすることができました。これで、あなたのコンピュータはバージョン管理を行う準備が整いました。

次にやるべきことは、Gitに対して、「**このコンピュータでGitを使うのは、私です**」と、自己紹介をすることです。

なぜ、自己紹介が必要なのでしょうか？

思い出してください。バージョン管理の大きなメリットの一つは、「いつ、**誰が**、なぜ変更したのか」を正確に記録することでした。Gitは、あなたが変更を記録（コミット）するたびに、その記録に「これは〇〇さんが行いました」という**署名**を付け加えます。チームで開発している時に、この署名がなければ、誰がどのコードを書いたのか分からなくなってしまいます。

このセクションでは、その「署名」として使われる、あなたの名前とメールアドレスをGitに設定します。これは、Gitを使い始める前に、**必ず一度だけ**行わなければならない、非常に重要な設定です。

---

## 詳細解説

Gitの設定は、`git config` というコマンドを使って行います。このコマンドには、設定の範囲を指定するためのオプションがあります。

*   `--global`: コンピュータ全体の設定。このコンピュータ上の、全てのGitリポジトリに適用されます。
*   `--local`: 特定のリポジトリのみの設定。そのリポジトリの `.git/config` ファイルに保存されます。（今回は使いません）

今回は、あなたのコンピュータ全体で共通して使われるユーザー情報を設定したいので、`--global` オプションを使います。

### 👤 ユーザー名の設定

まず、あなたの名前をGitに教えます。ターミナル（またはコマンドプロンプト）を開き、以下のコマンドを実行してください。`"Your Name"` の部分は、ご自身の名前に置き換えてください。名前は、半角のアルファベットで、ローマ字表記にするのが一般的です。

```bash
# "Your Name" の部分を、ご自身の名前に置き換えてください
git config --global user.name "Taro Yamada"
```

**ポイント**: 名前にスペースが含まれる場合は、必ずダブルクオーテーション `"` で囲ってください。

### 📧 メールアドレスの設定

次に、あなたのメールアドレスを設定します。このメールアドレスは、後ほど作成する**GitHubアカウントで登録するものと同じ**にすることを、強く推奨します。これにより、GitHub上であなたのコミットが、正しくあなたのアカウントと紐付けられます。

```bash
# "your_email@example.com" の部分を、ご自身のメールアドレスに置き換えてください
git config --global user.email "taro.yamada@example.com"
```

### ✅ 設定内容の確認

さて、本当に正しく設定できたか、確認してみましょう。以下のコマンドを実行すると、現在設定されている内容の一覧が表示されます。

```bash
# 設定一覧を表示
git config --list
```

実行結果の中に、以下のような行が含まれていることを確認してください。（他にもいくつかの設定項目が表示される場合があります）

```
user.name=Taro Yamada
user.email=taro.yamada@example.com
```

もし、名前やメールアドレスを間違えてしまった場合は、心配ありません。もう一度、同じ `git config` コマンドを実行すれば、新しい内容で上書きされます。

### 🎨 (おまけ) コマンドの出力を色付けする

必須ではありませんが、以下の設定も行っておくと、`git status` や `git log` などのコマンドの出力が色付けされ、非常に見やすくなるので、おすすめです。

```bash
git config --global color.ui auto
```

最近のGitでは、この設定がデフォルトで有効になっている場合も多いです。

## ✨ まとめ

このセクションでは、Gitを使い始めるための、最初の、そして最も重要な初期設定を行いました。

*   **なぜ設定が必要か**: Gitが変更を記録する際に、「誰が」行ったのかを明確にするための「署名」として、ユーザー名とメールアドレスが必要である。
*   **設定コマンド**: `git config --global` を使って、コンピュータ全体に適用される設定を行う。
    *   `git config --global user.name "Your Name"`
    *   `git config --global user.email "your_email@example.com"`
*   **設定の確認**: `git config --list` コマンドで、設定内容の一覧を確認できる。
*   **メールアドレスの注意点**: GitHubアカウントで使う予定のメールアドレスと、同じものを設定することが強く推奨される。

これで、Gitという「道具」に、あなたの魂を吹き込むことができました。Gitは、これからあなたが行う全ての変更を、あなたの名前と共に、忠実に記録し続けてくれます。
