# Tutorial 4-2-1: Gitのインストール

## 🎯 このセクションで学ぶこと

*   お使いのOS（WindowsまたはMac）に、Gitをインストールする方法を学ぶ。
*   ターミナル（コマンドプロンプト）で、Gitが正しくインストールされたことを確認する方法を学ぶ。

---

## 導入

前のチャプターで、GitとGitHubの強力な概念を学びました。いよいよ、その第一歩として、あなたのコンピュータにGitという「道具」をインストールします。

Gitのインストールは、あなたがエンジニアとしての道を歩み始めるための、重要な通過儀礼の一つです。ここでの手順は、OS（お使いのコンピュータの基本ソフト）によって少し異なりますが、どちらも非常に簡単です。画面の指示に従って、一つずつ、落ち着いて進めていきましょう。

このセクションを終えれば、あなたのコンピュータは、プロの開発者と同じように、バージョン管理を行うための基本的な能力を持つことになります。

---

## 詳細解説

### 🖥️ Macの場合

実は、最近のMacには、最初からGitがインストールされている場合がほとんどです。まずは、すでにインストールされていないか、確認してみましょう。

1.  **ターミナルを開く**: `command + space` でSpotlight検索を開き、「ターミナル」と入力して、Enterキーを押します。

2.  **Gitのバージョンを確認する**: ターミナルで、以下のコマンドを入力し、Enterキーを押します。

    ```bash
    git --version
    ```

    `git version 2.xx.x (Apple Git-xxx)` のように、バージョン番号が表示されれば、あなたのMacにはすでにGitがインストールされています。この先のインストール作業は不要です。おめでとうございます！

    もし、`command not found` というエラーが表示されたり、「コマンドライン・デベロッパ・ツールが必要です」という趣旨のダイアログが表示された場合は、以下の手順に進んでください。

3.  **Homebrewをインストールする (推奨)**: Homebrewは、Mac用のパッケージマネージャー（ソフトウェアを簡単にインストール・管理するためのツール）です。まだインストールしていない場合は、今後の開発で非常に役立つので、この機会にインストールしましょう。

    ターミナルで、以下のコマンドをコピー＆ペーストし、Enterキーを押します。（このコマンドは、Homebrewの公式サイトで最新のものを確認してください）

    ```bash
    /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
    ```

    インストール中に、何度かパスワードの入力や、Enterキーを押すことを求められます。画面の指示に従ってください。

4.  **Homebrewを使ってGitをインストールする**: Homebrewのインストールが終わったら、以下のコマンドでGitをインストールします。

    ```bash
    brew install git
    ```

5.  **再度バージョンを確認する**: インストール完了後、もう一度 `git --version` を実行し、バージョン番号が表示されることを確認してください。

### 💻 Windowsの場合

Windowsでは、**Git for Windows** という公式パッケージをインストールするのが、最も一般的で簡単な方法です。

1.  **公式サイトにアクセスする**: Webブラウザで、以下のGit公式サイトにアクセスします。
    
    [https://git-scm.com/](https://git-scm.com/)

2.  **インストーラーをダウンロードする**: トップページに表示されている、「Downloads for Windows」ボタンをクリックします。あなたのPC（64-bit版が一般的）に合ったインストーラーが、自動的にダウンロードされます。

    <!-- ここにスクリーンショットを挿入: Git公式サイトのダウンロードボタン -->

3.  **インストーラーを起動する**: ダウンロードした `.exe` ファイルをダブルクリックして、インストーラーを起動します。

4.  **インストーラーの設定を進める**: インストーラーを起動すると、たくさんの設定項目を質問されますが、**基本的には、全てデフォルト設定のまま「Next」を連打して問題ありません。** 初学者が設定を変更する必要がある項目は、特にありません。

    以下に、いくつかの主要な設定画面の簡単な説明を載せますが、よく分からなければ、気にせず「Next」進んでください。

    *   **Select Components**: インストールするコンポーネントを選びます。デフォルトのままでOKです。
    *   **Choosing the default editor used by Git**: Gitがコミットメッセージなどを編集する際に使うエディタを選びます。デフォルトは `Vim` というエディタになっていますが、もしあなたがVSCodeを主に使うのであれば、プルダウンメニューから `Visual Studio Code` を選ぶと、後々便利です。（しかし、分からなければデフォルトのままで全く問題ありません）
    *   **Adjusting your PATH environment**: `Git from the command line and also from 3rd-party software` (推奨) が選択されていることを確認します。これにより、コマンドプロンプトやPowerShellなど、どこからでも `git` コマンドが使えるようになります。

    全ての選択が終わったら、「Install」ボタンをクリックして、インストールを開始します。

5.  **インストール完了**: インストールが完了したら、「Finish」ボタンをクリックして、インストーラーを終了します。

### ✅ インストールの最終確認 (Windows)

1.  **コマンドプロンプトを起動する**: スタートメニューで「cmd」と検索し、コマンドプロンプトを起動します。

2.  **Gitのバージョンを確認する**: コマンドプロンプトで、以下のコマンドを入力し、Enterキーを押します。

    ```bash
    git --version
    ```

    `git version 2.xx.x.windows.x` のように、バージョン番号が表示されれば、インストールは成功です。おめでとうございます！

## ✨ まとめ

このセクションでは、お使いのコンピュータにGitをインストールし、コマンドラインから利用できることを確認しました。

*   **Mac**: 
    *   多くの場合、最初からインストールされている。
    *   `git --version` で確認し、なければHomebrew (`brew install git`) でインストールするのが推奨される。
*   **Windows**: 
    *   公式サイトから **Git for Windows** のインストーラーをダウンロードする。
    *   インストーラーの設定は、**全てデフォルトのまま「Next」**で問題ない。
*   **最終確認**: 
    *   ターミナル（またはコマンドプロンプト）で `git --version` を実行し、バージョン番号が表示されることを確認する。

これで、あなたのコンピュータは、Gitという強力な「バージョン管理の道具」を手に入れました。しかし、道具は手に入れただけでは意味がありません。次のセクションでは、この道具を使う「あなた」自身の情報を、Gitに登録する「初期設定」を行います。
