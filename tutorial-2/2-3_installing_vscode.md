# Tutorial 2-3: テキストエディタ（VSCode）のインストール

## 🎯 このセクションで学ぶこと

*   なぜプロの開発者が、メモ帳ではなくVSCodeのような高機能なテキストエディタを使うのかを理解する。
*   お使いのPC（WindowsまたはMac）に、Visual Studio Code（VSCode）をインストールする手順を学ぶ。
*   VSCodeを起動し、最初の画面構成を理解する。

---

## 導入

Webブラウザという「お皿」の準備ができました。次は、プログラミングという料理の最も中心的な作業、レシピ（コード）を書くための「**包丁とまな板**」、すなわち**テキストエディタ**を準備しましょう。

このコースでは、現代のWeb開発における「最強の包丁」とも言える、**Visual Studio Code（VSCode）** を使用します。

「テキストエディタって、ただ文字が打てるソフトでしょ？」
「Windowsのメモ帳と何が違うの？」

そう思うかもしれません。しかし、VSCodeはただのメモ帳ではありません。それは、料理人が最高のパフォーマンスを発揮するために、人間工学に基づいて設計された、切れ味抜群の包丁セットのようなものです。一度この切れ味を知ってしまうと、もう普通のメモ帳には戻れなくなるでしょう。

このセクションでは、VSCodeがなぜそれほどまでに開発者に愛されているのか、その魅力に触れながら、実際のインストール手順を丁寧に解説していきます。

---

## 概念の説明

### 🔑 なぜ、ただのメモ帳ではダメなのか？

プログラミングは、一文字でも間違えれば、全く動かなくなってしまう、非常に繊細な作業です。何千行、何万行にもなるコードの中から、たった一つのタイプミスを探し出すのは、至難の業です。

高機能なテキストエディタ（別名: **コードエディタ**）は、こうしたミスを未然に防ぎ、開発者がより快適に、そして創造的な作業に集中できるように、様々な補助機能を提供してくれます。

| 機能 | 説明 | メモ帳の場合 | VSCodeの場合 |
| :--- | :--- | :--- | :--- |
| **シンタックスハイライト** | コードの種類に応じて、文字に色を付けてくれる機能。 | 全て黒一色。 | **キーワードは青**、**文字列は赤**など、見やすく色分けしてくれる。 |
| **コード補完（サジェスト）** | コードの書き出しを打つと、候補を予測して表示してくれる機能。 | なし。全て手打ち。 | `fun`と打つだけで`function`の候補を出してくれる。 |
| **エラーチェック** | コードを保存する前に、文法的な間違いを指摘してくれる機能。 | なし。実行して初めてエラーに気づく。 | 間違っている箇所に、リアルタイムで**赤い波線**が表示される。 |
| **ターミナル統合** | コードを書く画面内で、コマンドを実行できる機能。 | 不可。別のウィンドウを開く必要がある。 | 画面を切り替えずに、シームレスにコマンド操作ができる。 |

これらの機能のおかげで、開発者は「コードのタイプミス」のような単純作業から解放され、「どんな機能を作るか」という、より本質的な問題解決に集中することができるのです。

## 詳細解説

それでは、実際にVSCodeをインストールしていきましょう。手順はWindowsとMacでほとんど同じです。

### ⚙️ VSCodeのインストール手順

1.  **インストーラのダウンロード**
    *   前のセクションでインストールしたGoogle Chromeを開き、検索窓で「`vscode`」と検索するか、以下のURLにアクセスします。
    *   **Visual Studio Code公式サイト**: [https://code.visualstudio.com/](https://code.visualstudio.com/)
    *   トップページに、あなたのOS（WindowsまたはMac）に合わせたダウンロードボタンが大きく表示されています。この青いボタンをクリックしてください。
    *   （ここにスクリーンショットを挿入: VSCode公式サイトのダウンロードボタン）

2.  **インストーラの実行（Windowsの場合）**
    *   ダウンロードフォルダに `VSCodeUserSetup-x64-xxxx.exe` というファイルが保存されます。これをクリックして実行します。
    *   使用許諾契約の画面が表示されたら、「同意する」にチェックを入れて「次へ」をクリックします。
    *   その後、いくつか設定画面が表示されますが、基本的には**全てデフォルトのまま「次へ」をクリック**して問題ありません。特に、「追加タスクの選択」画面では、全てのチェックボックスにチェックが入っていると、後々便利です。
    *   「インストール」ボタンをクリックすると、インストールが始まります。
    *   （ここにスクリーンショットを挿入: 使用許諾契約、追加タスクの選択、インストール開始画面）

3.  **インストーラの実行（Macの場合）**
    *   ダウンロードフォルダに `VSCode-Universal-stable.zip` というファイルが保存されます。これをダブルクリックして解凍します。
    *   解凍すると、`Visual Studio Code.app` というアプリケーションができます。これを、Finderの「アプリケーション」フォルダにドラッグ＆ドロップしてください。
    *   （ここにスクリーンショットを挿入: 解凍後のappファイルと、アプリケーションフォルダへの移動）

4.  **VSCodeの起動と初期設定**
    *   インストールが完了したら、VSCodeを起動してみましょう。
    *   初回起動時には、まず「ようこそ」タブが表示され、最初にいくつかの基本設定を尋ねられます。
    *   **テーマの選択**: まず、エディタの見た目（テーマ）を選びます。目に優しい「Dark Modern」（ダークモード）が人気ですが、これは後からいつでも変更できるので、好みで選んでください。
    *   **UI言語の変更**: 最初はUIが英語になっているかもしれません。左側のアクティビティバー（アイコンが縦に並んでいる場所）の一番下の「拡張機能」アイコン（四角が4つ並んだような形）をクリックします。検索窓に「`Japanese Language Pack`」と入力し、インストールボタンをクリックします。インストール後、右下に表示される「Change Language and Restart」ボタンを押すと、UIが日本語化されて再起動します。
    *   （ここにスクリーンショットを挿入: ようこそ画面、テーマ選択、拡張機能の検索とインストール、日本語化の再起動ボタン）

### ⚙️ 最初の画面構成

日本語化が完了し、再起動すると、以下のような画面が表示されます。これが、これから長い付き合いになるVSCodeの基本画面です。

（ここにスクリーンショットを挿入: 日本語化されたVSCodeの全体画面と、各部の名称を指し示す矢印）

| 番号 | 名称 | 役割 |
| :--- | :--- | :--- |
| 1 | **エディタ領域** | 最も大きな中央のエリア。実際にコードを書く場所です。 |
| 2 | **アクティビティバー** | 左端のアイコンが並ぶエリア。エクスプローラー、検索、Git、拡張機能など、主要な機能への入り口です。 |
| 3 | **サイドバー** | アクティビティバーで選択した機能の詳細が表示されるエリア。ファイル一覧（エクスプローラー）などがここに表示されます。 |
| 4 | **パネル** | 画面下部のエリア。ターミナル（コマンド実行画面）や、エラー一覧などがここに表示されます。 |
| 5 | **ステータスバー** | 最下部の青いバー。現在のブランチ名や、エラーの数など、様々なステータス情報が表示されます。 |

最初は覚える必要はありません。「コードは真ん中に書くんだな」「ファイルを探すときは左側を見るんだな」くらいの理解で十分です。使いながら、自然と覚えていきます。

## ✨ まとめ

このセクションでは、開発の「包丁とまな板」となるテキストエディタ、Visual Studio Codeのインストールと、基本的な設定を行いました。

*   **なぜVSCodeか？**: コードの色分け、入力補完、エラーチェックなど、快適な開発を支援する機能が満載だから。
*   **インストール手順**: 公式サイトからインストーラをダウンロードし、画面の指示に従ってインストール。日本語化パックを入れると、より使いやすくなる。
*   **基本画面**: 中央の「エディタ領域」でコードを書き、左の「サイドバー」でファイルを管理する、という基本構成を理解する。

これで、プログラミングという料理をするための、最高の「包丁とまな板」が手に入りました。次のセクションでは、この包丁をさらに使いやすくするための、便利な設定や拡張機能について見ていきましょう。
