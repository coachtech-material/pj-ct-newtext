# Tutorial 2-4: VSCodeの基本設定と拡張機能

## 🎯 このセクションで学ぶこと

*   VSCodeの使い勝手を向上させる、いくつかの重要な基本設定を理解し、適用する。
*   開発効率を劇的に向上させる、おすすめの「拡張機能」をインストールする。
*   設定ファイル（`settings.json`）の役割を知り、基本的な編集方法を学ぶ。

## 導入

最高の「包丁とまな板」であるVSCodeの準備ができました。しかし、買ってきたばかりの包丁が、必ずしも自分の手に完全にフィットしているとは限りませんよね。

プロの料理人が、自分の使いやすいように包丁の柄を調整したり、砥石で切れ味を調整したりするように、私たちもVSCodeを「**自分だけの最強の道具**」にカスタマイズしていきましょう。

このセクションでは、

2.  **拡張機能のインストール**: VSCodeの真骨頂である「拡張機能」の中から、このコースで特に役立つ、選りすぐりのものをインストールします。

これらの設定を行うことで、あなたのVSCodeは、ただのテキストエディタから、開発のあらゆる側面をサポートしてくれる、頼もしい「相棒」へと進化します。

## 概念の説明

### 🔑 設定: UIからの変更とJSONファイルでの管理

VSCodeの設定は、大きく分けて2つの方法で行うことができます。

1.  **UI（グラフィカルユーザーインターフェース）からの設定**: マウスで操作できる、分かりやすい設定画面から変更する方法です。初心者にはこちらがおすすめです。
2.  **`settings.json` ファイルでの設定**: 設定内容が記述された、`settings.json` という専用のJSONファイルを直接編集する方法です。より高度で細かい設定や、設定内容のバックアップ・共有に適しています。

UIから設定を変更すると、VSCodeが裏側で自動的に `settings.json` を書き換えてくれています。つまり、どちらの方法を使っても、最終的に行われていることは同じです。

このセクションでは、まずUIからの簡単な設定方法を学び、最後に `settings.json` がどのようなものかを見てみることにします。

### 🔑 拡張機能（Extensions）: VSCodeを"育てる"

拡張機能は、VSCodeに**後から追加できる、様々な便利機能のパッケージ**です。スマートフォンの「アプリ」を追加していく感覚に似ています。

世界中の開発者が、便利な拡張機能を開発・公開しており、そのほとんどが無料で利用できます。これらをインストールすることで、VSCodeは無限の可能性を秘めたツールになります。

*   特定のプログラミング言語（PHP, Python, etc.）のサポートを強化する
*   コードの見た目を自動で整えてくれる
*   Gitの操作をよりグラフィカルに行えるようにする
*   エディタの配色テーマやアイコンを追加する

拡張機能を追加していく行為は、まさに自分の相棒を「育てる」ような楽しさがあります。ただし、何でもかんでも入れれば良いというものでもありません。入れすぎると、かえって動作が重くなることもあります。まずは、定番で評価の高いものから試していくのが良いでしょう。

## 詳細解説

それでは、具体的な設定と拡張機能のインストールを進めていきましょう。

### ⚙️ 1. 基本設定の変更

まず、VSCodeの基本的な使い勝手を良くするための設定を行います。

左下の歯車アイコンをクリックし、「設定」を選択するか、ショートカットキー（`Ctrl + ,` on Windows, `Cmd + ,` on Mac）で設定画面を開きます。

（ここにスクリーンショットを挿入: 歯車アイコンから「設定」を開く様子）

検索窓に、以下の設定項目名を入力して、値を変更していきましょう。

| 設定項目 | 推奨値 | 説明 |
| :--- | :--- | :--- |
| `Editor: Font Size` | `16` or `18` | エディタの文字の大きさです。小さすぎると目が疲れるので、少し大きめが見やすいです。 |
| `Editor: Tab Size` | `4` | Tabキーを押したときに、どれくらい字下げ（インデント）するかの設定。PHPでは4が標準です。 |
| `Editor: Render Whitespace` | `boundary` | スペースやタブなどの「空白文字」を、薄い点で表示する設定。不要な空白を見つけやすくなります。 |
| `Files: Auto Save` | `onFocusChange` | ファイルの内容を変更した後、別のファイルやウィンドウに移動した際に、自動で保存してくれる設定。保存忘れを防げます。 |
| `Format On Save` | `true` (チェックを入れる) | ファイルを保存したタイミングで、後述するフォーマッタ（コード整形ツール）を自動で実行してくれる設定。 |

（ここにスクリーンショットを挿入: 設定画面で `Font Size` を検索し、変更している様子）

### ⚙️ 2. おすすめ拡張機能のインストール

次に、開発効率を格段に上げてくれる、おすすめの拡張機能をインストールします。

左側のアクティビティバーから「拡張機能」アイコンをクリックし、検索窓に以下の拡張機能名（ID）を入力して、一つずつインストールしていきましょう。

（ここにスクリーンショットを挿入: 拡張機能の検索窓にIDを入力している様子）

#### 必須レベル

| 拡張機能名 | ID | 説明 |
| :--- | :--- | :--- |
| **PHP All-in-One** | `onecentlin.laravel-blade` | PHPのコード補完、定義ジャンプ、エラーチェックなどを大幅に強化してくれます。PHP開発には必須です。 |
| **Laravel Blade Snippets** | `onecentlin.laravel-blade-snippets` | Laravelのテンプレートエンジン「Blade」の記法を、簡単に入力できるようにしてくれます。 |
| **Prettier - Code formatter** | `esbenp.prettier-vscode` | コードの見た目（インデント、改行など）を、設定したルールに従って自動で整形してくれるツール。チーム開発の必需品。 |
| **GitLens — Git supercharged** | `eamodio.gitlens` | コードの各行が「いつ」「誰によって」変更されたのかを表示してくれる拡張機能。Gitを使った開発が非常にやりやすくなります。 |
| **Thunder Client** | `rangav.vscode-thunder-client` | VSCode内でAPIのリクエストを送信し、レスポンスを確認できるツール。API開発のデバッグに絶大な威力を発揮します。 |

#### あると便利

| 拡張機能名 | ID | 説明 |
| :--- | :--- | :--- |
| **Material Icon Theme** | `PKief.material-icon-theme` | ファイルの種類に応じて、エクスプローラーのアイコンを見やすく、カラフルにしてくれます。 |

### ⚙️ 3. ターミナルからVSCodeを開く（`code .`コマンド）

開発作業では、ターミナルでフォルダを作成・移動した後、そのフォルダをVSCodeで開くという操作を頻繁に行います。このとき、`code .`コマンドを使うと、ターミナルから直接VSCodeを起動できます。

#### 「`code`」コマンドのインストール

**Windowsの場合**

VSCodeをインストールする際に、「PATHへの追加」オプションにチェックを入れていれば、自動的に`code`コマンドが使えるようになっています。使えない場合は、VSCodeを再インストールし、「PATHへの追加」オプションにチェックを入れてください。

**Macの場合**

1. VSCodeを開く
2. `Cmd + Shift + P`でコマンドパレットを開く
3. `Shell Command: Install 'code' command in PATH`を検索して実行
4. ターミナルを再起動

（ここにスクリーンショットを挿入: コマンドパレットでShell Commandを検索している様子）

#### `code .`コマンドの使い方

ターミナルでプロジェクトフォルダに移動した後、以下のコマンドを実行します。

```bash
# 現在のディレクトリをVSCodeで開く
code .
```

`.`（ドット）は「現在のディレクトリ」を意味します。このコマンドを実行すると、VSCodeが起動し、現在のディレクトリがプロジェクトとして開かれます。

> 💡 **ポイント**: 今後のハンズオンでは、フォルダを作成・移動した後に`code .`コマンドを実行する指示が頻繁に出てきます。このコマンドを覚えておきましょう。

### ⚙️ 4. 設定ファイル（settings.json）を覔いてみる

最後に、これまでUIで行ってきた設定が、裏側でどのように保存されているかを見てみましょう。

設定画面の右上にある、書類に矢印がついたようなアイコン（「設定(JSON)を開く」）をクリックしてください。

（ここにスクリーンショットを挿入: 設定画面右上のJSONを開くアイコン）

すると、`settings.json` というファイルが開きます。中には、先ほど設定した項目が、以下のような形式で記述されているはずです。

```json
{
    "editor.fontSize": 16,
    "editor.tabSize": 4,
    "editor.renderWhitespace": "boundary",
    "files.autoSave": "onFocusChange",
    "editor.formatOnSave": true
}
```

*   `"editor.fontSize": 16,` のように、`"設定項目名": 値,` というキーと値のペアで構成されています。
*   このファイルを直接編集して保存することでも、設定を変更できます。
*   チームで開発する際は、この `settings.json` の一部を共有することで、全員の開発環境を統一することができます。

今は、「こういうファイルで管理されているんだな」という程度の理解で大丈夫です。
